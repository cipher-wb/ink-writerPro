## Codebase Patterns
(承接自前轮 ralph runs，保留跨 PRD 通用条目。逐轮新增追加到下方。)

- **UTF-8 编码硬约束**：所有 `open()` / `Path.read_text()` / `Path.write_text()` 必须带 `encoding="utf-8"`。scripts/fix_utf8_encoding.py 可批量修复；scripts/audit_cross_platform.py 是 9 类风险单一扫描入口。
- **Windows 兼容守则**：新增 .sh 必须配 .ps1（UTF-8 BOM）+ .cmd 双击包装；新增带 `if __name__ == "__main__":` 的脚本在 main 开头调用 `runtime_compat.enable_windows_utf8_stdio()`（Mac no-op）。详见 CLAUDE.md。
- **agent 规格统一目录**：所有 agent 规格在 `ink-writer/agents/`（非 `ink_writer/`，US-402 已消除双目录）。
- **pytest.ini testpaths 必须显式注册**：新建 `tests/<module>/` 目录后必须追加到 `pytest.ini` 的 `testpaths` 行，否则 pytest 不收集；项目覆盖率门禁 `--cov-fail-under=70`。运行时可用 `pytest tests/<new-dir>` 直接目标调用绕过 testpaths，适合 pre-US-002 的单任务验证。
- **Python 包 vs 插件目录命名**：`ink_writer/` 是 Python 包（pyproject.toml 注册），`ink-writer/` 是 Claude Code 插件（agents / skills / scripts）；不可混用。
- **Ruff 是 lint gate**（`python3 -m ruff check`）。常见自动修复：UP009（去 `# -*- coding: utf-8 -*-`）、UP035（`typing.Sequence` → `collections.abc.Sequence`）、I001（import ordering）。新增 .py 提交前先跑 `ruff check --fix <paths>`。
- **无 mypy/pyright 配置** — PRD "Typecheck passes" 的实际含义：`python3 -m py_compile <file>` + `ruff check` 同时干净。
- **断链检测标准写法**：`p.is_symlink() and not p.exists()`（`Path.exists()` 会 follow 符号链接，所以返 False 即 target 丢失）。
- **仓库级 audit 红线会扫全仓 .py**：`tests/audit/test_cli_entries_utf8_stdio.py::test_repo_has_no_cli_entry_missing_utf8_stdio`（任何含 `__main__` 的 .py 必须出现 `enable_windows_utf8_stdio` 字面量）与 `tests/core/test_safe_symlink.py::test_repo_has_no_bare_symlink_calls_outside_primitives`（禁止 `os.symlink` / `Path.symlink_to` 裸调用，须走 `runtime_compat.safe_symlink`；测试 fixture 处用 `# c5-ok: 原因` 豁免）。**新增任何 .py 之后一定要跑这两个测试**，不能只依赖子目录 pytest 绿灯。
- **PEP 440 `~=X.Y` 语义**：`>=X.Y, <(X+1).0`（允许次版本号上升到 `(X+1).0` 之前），不是"只允许 X.Y.z"。要锁次版本号写 `~=X.Y.0`。PRD 里的"输出 1.12.x"类措辞是估值，判合规看约束不看措辞。

- **Learnings for future iterations:**
  - Pattern discovered: **`argparse.BooleanOptionalAction`**（Python 3.9+）是`--flag / --no-flag` pair 的标准写法——一次 `add_argument(..., action=BooleanOptionalAction, default=True)` 同时暴露两个 flag，比手动 `add_mutually_exclusive_group` + `store_true/store_false` 简洁 5 行。PRD 里"`--require-foo/--no-require-foo`（默认 True/False）"措辞直接落地成一行代码。未来 M2/M3 若有其他"启用 X / 禁用 X"型开关，一律用这个 action。
  - Pattern discovered: **CLI default ≠ module default** 是合理的双轨设计。`PreflightConfig.require_embedding_key: bool = False`（模块层，测试默认不要求）vs CLI `--require-embedding-key` default True（运维入口，默认严格）。**不要**为了"统一"把两者都改成同一值——测试方便和生产严格是两个 persona，各自取默认值就是最干净的。后续其他模块若遇到类似张力（如 M2 的 "--strict-mode default True"），照此双轨。
  - Pattern discovered: **"raise_on_fail=True 对 CLI 是陷阱"**——一旦 `run_preflight` 抛 PreflightError，CLI 再也拿不到逐项 CheckResult 打印了（错误对象只带 check_names，不带 detail）。正解是：CLI 始终以 `raise_on_fail=False` 调 `run_preflight`，打印完报告后自己判失败打 rc=1。`--raise-on-fail` flag 就退化为"多写一份 stderr 字面错误"——shell 调用方看 rc 就够。若将来要让 `PreflightError` 携带完整 report（改 errors.py 加 `results` 字段），这条 learning 也一并失效，可以改回直接 raise。当前为最小表面积变更选 pass-through。
  - Pattern discovered: **SKILL.md heading 层级是跨 skill 共享的树**——直插一条与父级同阶的 `##` 会让后面所有 `###` 变成新节点下的孙辈，污染整篇 skill 目录结构。遇到 PRD AC 写了一个层级、但现有文件已有不同层级时，**遵守"不打破 heading tree"的原则优于字面跟随**，并在 progress.txt 里显式记录偏差（本次用 `####` 代替 `##`）。下一轮 ralph 若 PRD 又写 `##`，用同样的解释即可。
  - Pattern discovered: **CLI 测试不要 `subprocess.run(["python3", "-m", ...])`**——会把 `encoding="utf-8"` 管道、coverage 插桩、`capsys` 机制统统打掉，调试极慢。直接 `from ink_writer.preflight.cli import main` + `main(argv)` + `capsys.readouterr()` 是 3x 快、10x 可靠。仅当测"`__main__` 入口能否被 `-m` 找到"时才需要 subprocess（此场景不测）。
  - Gotchas 1：**`argparse.SystemExit` 的 `.code` 可能是 `None` 或 str**（`parser.error("...")` 传的是 str）。本 CLI 用 `exc.code if isinstance(exc.code, int) else 2` 做防御——否则 rc 会变成非 int 导致 `sys.exit(rc)` 再触发 SystemExit 字符串。US-008 ink case CLI 已有同样处理，新 CLI 沿用。
  - Gotchas 2：**`[OK ]` 中间必须有一个空格**（4 字符等宽与 `[FAIL]`）——不要写成 `[OK]`（3 字符），否则视觉对齐炸掉，人类读报告时第一眼就难受。这个细节 PRD 其实写了："`[OK ] <name>: <detail> 或 [FAIL] <name>: <detail>`"——空格是契约。
  - Gotchas 3：**`data/editor-wisdom/rules.json` 生产路径确实有横杠不是下划线**——本 repo 现有数据目录是 `data/editor-wisdom/`（ekahtnuh 风），但 Python 包名 `ink_writer/editor_wisdom/`（snake_case）。CLI 默认路径要 track 生产数据目录名；不要为了"美观"统一成下划线，否则生产跑会读不到 rules.json。
  - Context：下一轮 ralph 应挑 **US-017**（M1 端到端集成测试 + 验收）。关键点：(1) 新增 `tests/integration/test_m1_e2e.py`（2 用例：preflight_fail_creates_infra_case_visible_via_cli / preflight_pass_creates_no_new_cases）；(2) **不**用 `subprocess.run`——跨 CLI 测试用 `from ink_writer.preflight.cli import main` + `from ink_writer.case_library.cli import main as case_main` + `capsys`，延续本次 learning；(3) 核对 6 项验收（pytest ≥70 覆盖率 / Qdrant readyz 200 / fix_reference_corpus_symlinks missing=0 / preflight CLI 全过 / init_zero_case 输出 / grep SKILL.md 命中）；(4) 最后打 `git tag m1-foundation -m "M1 complete: case_library + qdrant + preflight + symlink fix"`；(5) **完成 US-017 后触发 ROADMAP 推进**——memory 里 `project_quality_overhaul_roadmap.md` 明确要求"M1 完成后必须主动推进 M2"，不能停下。
