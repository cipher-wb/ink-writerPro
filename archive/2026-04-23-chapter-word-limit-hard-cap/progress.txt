  - Shell 内嵌 Python 派生配置值时，一定要在 bash 侧再做一层正则 `[0-9]+` 校验——Python 侧任何报错（包括 stderr 被意外 merge 到 stdout）都可能污染 `$(...)` 输出，让 `(( ... ))` 触发 set -euo pipefail 下的 fatal。当前实现用 `|| echo 5000` + `[[ =~ ]]` 双兜底，够稳。
  - PowerShell `ConvertFrom-Json` 访问 `$prefData.pacing.chapter_words` 时若上层缺字段会直接 null dereference；`Set-StrictMode -Version Latest` 下更凶。目前用 `if ($prefData -and $prefData.pacing -and $prefData.pacing.chapter_words)` 三层 null-gate；若以后字段名/层级调整，记得全部 null-gate 一起改。
  - 抠 bash function 做黑盒测试（`_extract_bash_function`）是 ink-auto 测试的通用模式（US-012 起已用）；新加 function 要保证 `^fn_name() {` 独占一行 + `\n}\n` 收尾，否则正则抠不到。
  - 用 `ascii 'a' * (N-1) + '\n'` 可精确让 `wc -m` 返回 N（纯 ASCII 文件 char == byte）；CJK 测试需要用 UTF-8 字节/字符区分的场景另议。
  - 重试循环在 `set -uo pipefail`（无 `-e`）的主脚本里必须每个 bash 循环变量都预先显式初始化（`RETRY_ROUND=0` / `RETRY_VERIFIED=0`），否则 `while (( ... ))` 碰到未定义变量会被 `-u` 杀。
