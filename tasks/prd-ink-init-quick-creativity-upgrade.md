# PRD: `/ink-init --quick` 创意生成升级

## 1. Introduction / Overview

`/ink-init --quick` 目前能一键生成 3 套网文方案（题材/主角/冲突/金手指/书名），但用户反馈**套路感依然过重**，尤其是金手指（多数是"修为暴涨""无限金币""系统签到"类战力/资源型），书名和人名也偏向"xx神帝""萧xx"这种被嚼烂的范式。

现有的 `references/creativity/` 下已有 anti-trope 各题材文件、creative-combination、inspiration-collection、selling-points 等基础设施，但它们**是被动参考资料**，在方案生成阶段没有形成硬约束，导致 LLM 回落到训练数据里最高频的套路模板。

本 PRD 的目标是把创意从"参考材料"升级为"硬约束 + 强制扰动 + 校验闭环"，让 `/ink-init --quick` 生成的 3 套方案在金手指创意、语言风格、书名/人名独特性上产生显著区分度，同时保留商业可读性的弹性——不是每套都要"野路子"，而是给用户明确的创意档位选择权。

---

## 2. Goals

- **G1**：金手指创意度显著提升——生成的金手指中，≥80% 命中「非战力维度 + 代价可视化 + 一句话爆点」三重约束里至少 2 项
- **G2**：书名/人名的套路化指数下降——随机抽样 30 个方案，"萧/林/叶 + 尘/风/寒/云"类陈词组合占比 ≤10%（当前估计 ≥50%）
- **G3**：语言风格辨识度——3 套方案在语言尺度上形成「文学狂野 / 烟火接地气 / 江湖野气」三档鲜明区分，用户一眼能区分
- **G4**：创意多样性可控——新增激进度档位（保守/平衡/激进），用户可按需求选档，而非每次都赌随机
- **G5**：创意不与商业性对立——每套方案仍需通过 `selling-points.md` 的卖点校验，不能为创意牺牲可写可读性
- **G6**：鲜度保障——每次 quick 联网拉取最新榜单套路，确保反套路建立在"知道当下主流是什么"的基础上（带缓存防止延迟爆炸）

---

## 3. User Stories

### US-001：创意元规则库（Layer 1）

**Description**：作为创作者，我需要一份抽象的「创意元规则」，让系统在生成方案时有稳定的创意检验标尺，而非随机拼贴。

**Acceptance Criteria**：
- [ ] 新建 `ink-writer/skills/ink-init/references/creativity/meta-creativity-rules.md`
- [ ] 至少包含 10 条元规则，每条含：规则名 / 核心定义 / 正例 2 个 / 反例 2 个 / 可命中的题材范围
- [ ] 初版规则集必须覆盖：M01 代价可视化 / M02 反向冲突 / M03 信息不对称 / M04 角色错位 / M05 时空错位 / M06 身份反差 / M07 欲望悖论 / M08 系统反噬 / M09 权力颗粒度 / M10 尺度跃迁
- [ ] 每条规则标注「适配题材」（仙侠/都市/末世/科幻/悬疑/游戏/历史/现实/恋爱全覆盖）
- [ ] SKILL.md Quick Step 0 加载此文件为 L1 必读

### US-002：反套路种子库（Layer 2）—— 一次性 1000 条

**Description**：作为创作者，我需要一个预先写好的"稀缺元素组合池"，让生成方案时从中随机抽取具体素材强制混搭。本 PRD 实施阶段一次性产出 1000 条高质量种子，不再拆分到 3 个月后。

**Acceptance Criteria**：
- [ ] 新建 `ink-writer/skills/ink-init/references/creativity/anti-trope-seeds.json`
- [ ] 结构化字段：`seed_id / category / value / rarity（1-5） / genre_tags / example_pairing / source（llm/human/websearch_filtered）`
- [ ] **总量 1000 条**，覆盖 **10 个 category**，每类 100 条：
    - 基础 6 类：职业 / 时代 / 冲突 / 世界观 / 情感 / 禁忌
    - 扩展 4 类：神话 / 禁忌语言 / 身体特征 / 物件
- [ ] **稀有度分布硬约束**：
    - 稀有度 1（常见）：≤ 10%（最多 100 条，仅作基础打底）
    - 稀有度 2：≤ 20%（最多 200 条）
    - 稀有度 3：20%-30%（200-300 条）
    - 稀有度 4：≥ 30%（至少 300 条）
    - 稀有度 5（极反常规）：≥ 20%（至少 200 条）
- [ ] 种子示例必须"接地气但反常规"——如 `职业: 殡葬业主持人 / 网约车交警 / 二甲医院算命科医生 / 讨薪农民工头子`
- [ ] 配套提供 `anti-trope-seeds-schema.json` 用于 JSON 校验（字段类型、枚举值、稀有度范围）
- [ ] Typecheck（JSON 合法性）通过

### US-002b：种子库分批生成 + 人工 review 执行流程

**Description**：作为维护者，我需要一个可复现、可中断、可续跑的种子生成流程，保证 1000 条质量均衡，且每批 review 耗时可控。

**Acceptance Criteria**：
- [ ] **分 10 批执行**（每 category 一批 100 条），每批流程严格遵循：
    1. **Claude 生成 150 条候选**（多 50% 冗余用于淘汰）
    2. **WebSearch 反查**——并行搜索"起点 该类 热门"、"番茄 该类 爆款"，抽出 Top 20 高频元素作为该批黑名单
    3. **黑名单剔除**——候选中命中黑名单的自动删除
    4. **去重**——批内 + 与已入库种子的字面相似度（cosine，基于字符 bigram）>0.70 剔除
    5. **稀有度均衡补齐**——若稀有度 4+5 合计 <50 条则自动追加生成
    6. **随机抽样 20 条 review**——Claude 把这 20 条以 Markdown 列表贴在对话里，等用户逐条 ✅/❌ 打勾
    7. **用户反馈消化**——❌ 的种子 + 用户备注作为下一批的 negative prompt（"避免这种类型的"）
    8. **本批最终入库 100 条**（保留候选中 review 通过的 + 未抽中但无相似冲突的）
- [ ] **批次顺序**：职业 → 时代 → 冲突 → 世界观 → 情感 → 禁忌 → 神话 → 禁忌语言 → 身体特征 → 物件（排在前的 review 反馈影响后续批次）
- [ ] **中断续跑**：每批完成后立即写入 `anti-trope-seeds.json`，中断后重启从下一未完成批次继续
- [ ] **Review 输出格式固定**（贴在对话的模板）：
    ```markdown
    ### 批次 N：职业类 review（20/150 抽样）
    请逐条打勾：✅ 保留 / ❌ 剔除 / ⚠️ 需调整

    1. [ ] 殡葬业主持人（稀有度 4）—— 示例搭配：葬礼变相亲现场
    2. [ ] 网约车交警（稀有度 4）—— 示例搭配：副业是查自己乘客
    ...
    20. [ ] ...

    请回复如 `1✅ 2❌（太俗） 3⚠️改成"网约车检察官"` 或 `全部✅` 快速通过
    ```
- [ ] **Review 压力可控**：单批用户 review 时间 ≤ 8 min（20 条 × ~20s/条），10 批总 ≤ 80 min（允许分段做，每批完成自动暂停等用户）
- [ ] **质量指标**：所有批次合计用户 ✅ 率 ≥ 80%，否则说明生成规则有问题，需回看元规则库调整后重跑
- [ ] **生成日志**：`data/anti-trope-seeds-gen-log.md` 记录每批的候选数/黑名单剔除数/去重数/review 通过率/最终入库数

### US-003：随机扰动引擎（Layer 3）

**Description**：作为创作者，我需要一个强制混搭机制，每次 quick 调用时自动把稀缺元素拼到方案里，避免 LLM 回落到训练集高频套路。

**Acceptance Criteria**：
- [ ] 新建 `ink-writer/skills/ink-init/references/creativity/perturbation-engine.md`
- [ ] 定义「扰动对」概念——每套方案必须注入 ≥2 对扰动（如「古代+直播」「和尚+金融」「高中生+企业家」），扰动对从种子库随机抽取
- [ ] 定义 5 种扰动模式：时代错位 / 职业反差 / 尺度错位 / 规则颠倒 / 感官替换
- [ ] 每套方案在输出里必须明示使用的扰动对（便于用户理解创意来源）
- [ ] 3 套方案之间的扰动对完全不重复

### US-004：激进度档位前置选择（Quick Step 0.5）

**Description**：作为创作者，我希望在 quick 调用时能先选"创意激进度"，避免每次都要在随机结果里碰运气。参数用纯数字（1/2/3/4）避免拼错英文。

**Acceptance Criteria**：
- [ ] Quick Mode 在 Step 0 与 Step 1 之间插入 Step 0.5：激进度选择
- [ ] 提供 **4 档**：
    - **1 = 保守**（1-2 个反套路点，主流套路打底，商业安全）
    - **2 = 平衡**（3-4 个反套路点，商业+实验各半，默认档）
    - **3 = 激进**（5+ 个反套路点，以野路子为主）
    - **4 = 疯批**（完全不考虑商业性，纯实验，每套方案必须 ≥5 条元规则 + ≥5 对扰动对 + 稀有度 5 的种子强制多条）
- [ ] 交互使用 AskUserQuestion，选项为中文："1 保守 / 2 平衡 / 3 激进 / 4 疯批"
- [ ] 默认档位：**2 平衡**（用户直接回车或输入空时）
- [ ] **支持 3 种命令格式（用户只需记一种）**：
    - `/ink-init --quick`（不带档位，弹出询问）
    - `/ink-init --quick 2`（位置参数，推荐记这个最简洁）
    - `/ink-init --quick --档位=2`（显式命名参数，备用）
- [ ] 档位输入容错：用户输入"保守/平衡/激进/疯批"中文词也能识别，映射到对应数字档位
- [ ] 选定档位后，3 套方案均按该档位生成（不再做 3 档梯度分离）
- [ ] 输出日志显式标注本次使用的档位（如「本次生成档位：3 激进」）

### US-005：3 套方案按激进度档位生成

**Description**：作为创作者，我希望 3 套方案根据选定的激进度档位统一调整创意强度，不做档位交叉。

**Acceptance Criteria**：
- [ ] **档位 1 保守**：每套方案命中 ≥1 条元规则 + 1 对扰动对，扰动对稀有度 ≤3
- [ ] **档位 2 平衡**：每套方案命中 ≥2 条元规则 + 2 对扰动对，至少 1 对稀有度 ≥4
- [ ] **档位 3 激进**：每套方案命中 ≥3 条元规则 + 3 对扰动对，至少 2 对稀有度 ≥4
- [ ] **档位 4 疯批**：每套方案命中 ≥5 条元规则 + 5 对扰动对，至少 3 对稀有度 =5，且必须打破至少 1 个「商业安全边界」（如反派视角主角 / 悲剧结局主线 / 主角长期失败等）
- [ ] 3 套方案之间的元规则、扰动对、题材方向三维度两两不重复
- [ ] 生成时在 think 过程中自检规则命中数，未达标则重试（最多 3 次）

### US-006：金手指三重硬约束校验

**Description**：作为创作者，我希望金手指不再是"战力暴涨"或"系统签到"俗套，必须在非战力维度、代价可视化、一句话爆点三项里至少满足 2 项。

**Acceptance Criteria**：
- [ ] 新建 `ink-writer/skills/ink-init/references/creativity/golden-finger-rules.md`
- [ ] 定义 3 条硬约束：
      - **GF-1 非战力维度**：禁用纯力量/资源类，必须是信息/时间/情感/社交/认知/概率/感知/规则类之一
      - **GF-2 代价可视化**：必须有明确、可量化、可被反派利用的成本（如"每用一次少活 1 小时"、"只能对说过谎的人生效"）
      - **GF-3 一句话爆点**：20 字内能讲清楚且让人产生"这也行？"的第一反应
- [ ] 每个金手指输出时必须附 `gf_checks` 字段，标注 3 项通过情况（0/1）
- [ ] 3 项中至少 2 项为 1，才能通过校验
- [ ] 校验失败时重新生成（最多 5 次），仍失败则 fallback 到降档（如激进→平衡）

### US-007：语言风格三档分级 + 敏感词尺度

**Description**：作为创作者，我希望 3 套方案在语言风格上形成「文学狂野 / 烟火接地气 / 江湖野气」三档鲜明区分，书名/人名/简介都要风格化，且允许适度粗口/擦边暗示但不违规。

**Acceptance Criteria**：
- [ ] 新建 `ink-writer/skills/ink-init/references/creativity/style-voice-levels.md`
- [ ] 定义 3 档风格：
      - **V1 文学狂野**：意象粗粝有美感，如书名「赤地一拳」「霓虹佛陀」「刀山火海打菩萨」；人名可用生僻字但不怪异，如「池冷灰」「陆无归」
      - **V2 烟火接地气**：口语化方言感，如书名「老子就是救世主」「外卖送到阎王殿」「菜市场里修真人」；人名带烟火气，如「王满仓」「赵二娃」「孙秃子」
      - **V3 江湖野气**：带绰号感和轻度粗口，如书名「卧槽这关没打过」「神仙打屁」「断头饭吃到撑」；人名用绰号体，如「刀疤阿九」「臭嘴老七」「烂命三哥」
- [ ] **敏感词分级规则**（PRD 强制）：
      - 允许：卧槽 / 妈的 / 他娘的 / 滚蛋 / 去死 / 老子 / 丫的 / 龟儿子
      - 允许擦边暗示：「腰比蛇软」「眼神能勾人魂」「夜里不老实」
      - **禁止**：任何具体生殖器官名、具体性行为描写、具体脏字主体（操/屌/逼/屄）、辱骂特定族群/宗教
      - 每章粗口密度 ≤ 0.3%（约每 1000 字 3 个），避免刷屏式低俗
- [ ] 3 套方案分别对应 V1/V2/V3 一档（同一次 quick 内），形成对比
- [ ] **激进度档位影响粗口/擦边密度（允许档位间浮动）**：
    - 档位 1 保守：禁用 L1 粗口，全套方案粗口密度 0%
    - 档位 2 平衡：每套允许 2-3 处 L1 粗口（约 0.2%/千字）
    - 档位 3 激进：允许 L1 粗口密度 0.5%-0.8%/千字，允许 L2 擦边暗示每套 1-2 处
    - 档位 4 疯批：L1 粗口密度 0.8%-1.5%/千字，L2 擦边暗示每套 3-5 处，可尝试 L2 临界突破（但仍不触 L3 红线）
- [ ] 所有档位共同遵守 L3 红线（禁止具体生殖器官词、具体性行为描写、辱骂特定族群/宗教），无档位豁免

### US-008：书名/人名生成算法升级（含修辞标签）

**Description**：作为创作者，我希望书名和人名不再是「xx神帝/萧/林/叶 + 尘/风/寒/云」的陈词模板，而是根据风格档位动态组合，并引入修辞标签让书名更有匠人味。

**Acceptance Criteria**：
- [ ] 更新 `data/naming/given_names.json` 新增风格标签：`rough（粗粝）/ smoky（烟火）/ jianghu（江湖绰号）`
- [ ] 新建 `data/naming/nicknames.json` —— 江湖绰号库（刀疤/癞子/麻子/瘸子/秃子/哑巴/独眼/瞎子/烂命/臭嘴/二愣子等 100+ 条），每条含 `rarity` 和 `style_tags`
- [ ] 新建 `data/naming/book-title-patterns.json` —— 书名模板库，按 V1/V2/V3 三档各存 50+ 条结构化模板
- [ ] **书名模板新增修辞标签 `rhetoric_tags`（多选）**：
    - `pun` 双关（如《卧槽这关没打过》"关"双关）
    - `homophone` 谐音（如《姨父不懂事》谐音"遗腹")
    - `antithesis` 对仗（如《刀山火海打菩萨》）
    - `irony` 反讽（如《老子就是救世主》）
    - `oxymoron` 矛盾修辞（如《温柔的刽子手》）
    - `concrete_abstract` 具象配抽象（如《赤地一拳》）
    - `anachronism` 时空错置（如《外卖送到阎王殿》）
- [ ] 书名生成必须强制命中 ≥1 个修辞标签，同一次 quick 的 3 套方案书名修辞标签不重复
- [ ] 书名/人名生成禁用词（default blacklist 扩展）：
      - 书名禁用后缀："神帝/天帝/至尊/大帝/无敌/主宰/龙傲天/战神/剑神/丹帝/仙尊"
      - 书名禁用前缀："我的/全球/无敌/最强/重生成" + 职业 / 身份（如"我的美女老板"、"全球高武"类陈词）
      - 人名禁用："萧/林/叶/楚/顾/秦/陆" 作为姓 × "尘/风/寒/云/逸/辰/墨/天" 作为名的任意组合（全禁）
- [ ] 每次生成时抽样 10 次，"陈词组合"命中率 ≤10%（用单元测试覆盖）
- [ ] 单元测试还需验证：每次生成的 3 个书名修辞标签各异

### US-009：每次 Quick 联网拉取最新榜单（带缓存）

**Description**：作为创作者，我希望系统在生成方案前知道"当下主流套路是什么"，才能有效反向操作。**固定只搜起点和番茄**，不做可配置项（避免增加复杂度）。

**Acceptance Criteria**：
- [ ] Quick Step 0 新增 WebSearch 子步骤，**固定 2 个检索源**：
    - 起点中文网榜单（检索语：`起点中文网 月票榜 2026 热门 题材` + `起点 分类 月榜`）
    - 番茄小说（检索语：`番茄小说 爆款 套路 2026` + `番茄免费小说 热门 新书`）
- [ ] 检索源硬编码，不支持用户自定义检索语
- [ ] 本地缓存机制——`data/market-trends/cache-YYYYMMDD.md`，同一天多次 quick 调用共享缓存，跨天重新搜索

## 4. Functional Requirements

### 创意规则体系

- **FR-1**：系统必须加载 `meta-creativity-rules.md`（Layer 1）作为每次 quick 调用的硬约束检验集
- **FR-2**：系统必须加载 `anti-trope-seeds.json`（Layer 2）作为随机抽取的灵感源，**最少 1000 条种子，覆盖 10 个 category**，稀有度 ≥4 占比 ≥50%，稀有度 5 占比 ≥20%
- **FR-3**：系统必须执行扰动引擎（Layer 3），每套方案强制注入 ≥ 对应激进度档位的扰动对数量
- **FR-4**：3 套方案在题材方向、元规则、扰动对、语言档位四个维度上两两不重复

### 激进度档位

- **FR-5**：Quick Mode Step 0.5 必须询问用户激进度（AskUserQuestion），4 档选项「1 保守 / 2 平衡 / 3 激进 / 4 疯批」，默认 **2 平衡**
- **FR-6**：用户可通过 3 种方式跳过询问——位置参数 `/ink-init --quick 2`、命名参数 `/ink-init --quick --档位=2`、中文词 `/ink-init --quick 激进`
- **FR-7**：激进度影响四项：规则命中数下限 / 扰动对数量 / 扰动对稀有度下限 / 粗口擦边频率（见 US-005 和 US-007 的详细档位表）

### 金手指校验

- **FR-8**：每个金手指必须通过三重校验（GF-1/GF-2/GF-3），至少 2 项为 1
- **FR-9**：校验未通过时自动重新生成，最多 5 次，仍失败则降档一级（激进→平衡→保守）
- **FR-10**：金手指禁止词：修为暴涨 / 无限金币 / 系统签到 / 作弊器 / 外挂 / 无限奖励 等 20+ 个高频俗套词

### 语言风格

- **FR-11**：3 套方案必须分别分配 V1/V2/V3 三档语言风格，每套的书名、人名、简介都需贯彻该档风格
- **FR-12**：粗口/擦边使用须遵守 US-007 定义的敏感词分级（允许列表 + 禁止列表 + 密度上限）
- **FR-13**：书名禁止后缀 20+ 个俗套词（神帝 / 至尊 / 无敌 等），书名生成必须从 `book-title-patterns.json` 按档位抽取模板
- **FR-14**：人名禁止「萧/林/叶 × 尘/风/寒/云/逸/辰」等 20+ 个陈词组合（硬黑名单）

### 联网 & 缓存

- **FR-15**：Quick Step 0 执行 WebSearch 固定抓取起点中文网 + 番茄小说两个平台，检索语硬编码不可配置
- **FR-16**：缓存文件 `data/market-trends/cache-YYYYMMDD.md`，当日共享，跨日重新搜索，保留最近 90 天
- **FR-17**：每套方案必须规避两平台共通 Top 5 热门套路关键词（单平台榜单套路不强制规避），输出时显式标注规避项
- **FR-18**：WebSearch 失败时 fallback 到最近 7 天内缓存，并在输出中提示"使用 N 天前榜单数据"

### 输出格式

- **FR-19**：每套方案输出末尾必须包含「🧬 创意指纹」板块，含 5 字段（命中元规则 / 扰动对 / 金手指维度矩阵 / 语言档位 + 示例 / 反向规避项）
- **FR-20**：指纹板块总字数 ≤ 200 字，保持信息密度

## 6. Design Considerations

### 6.1 三层创意体系的分工

```
Layer 1 (元规则) —— 是什么算"不套路"的抽象定义（硬约束）
    ↓
Layer 2 (种子库) —— 提供具体的"反常规素材"（灵感源）
    ↓
Layer 3 (扰动引擎) —— 随机混搭机制（爆破器）
    ↓
生成方案 → 金手指校验 → 语言风格分配 → 输出 + 创意指纹
```

### 6.2 语言档位示例（3 套方案风格对比）

| 档位 | 书名示例 | 主角名示例 | 金手指一句话 |
|------|---------|-----------|-------------|
| V1 文学狂野 | 《赤地一拳》 | 池冷灰 | 能看见别人死亡时的最后一幅画面，画面越美他越想死 |
| V2 烟火接地气 | 《外卖送到阎王殿》 | 王满仓 | 外卖超时一分钟，阎王就少给他一天阳寿 |
| V3 江湖野气 | 《卧槽这关没打过》 | 刀疤阿九 | 他娘的赌神附体，但只能赌输不能赢——输一次家里死一个仇人 |

### 6.3 金手指反例（用于训练过滤）

以下类型强制拒绝：
- "修为一日千里" / "获得系统签到" / "无限金币奖励" / "觉醒吞噬天赋" / "外挂面板"
- "重生回到十年前" 单独作为金手指（如果无额外代价/维度约束则拒绝）
- "随便一拳打爆星球" / "纯战力暴涨类"

以下类型鼓励：
- "能听见纸币的记忆"（认知维度 + 代价：听多了会精神分裂）
- "别人夸他一句他瘦一斤"（社交维度 + 代价：瘦到 0 会消失）
- "预知自己下次挨打的时间和地点，但不能告诉别人"（信息维度 + 代价：说出口就失效）

### 6.4 敏感词分级执行策略

PRD 硬规则：
| 级别 | 示例 | 允许档位 | 密度上限 |
|------|------|---------|---------|
| L0 中性俚语 | 老子 / 丫的 / 瞎整 | V1/V2/V3 全通 | 无限制 |
| L1 轻度粗口 | 卧槽 / 妈的 / 他娘的 / 龟儿子 | V2/V3 | 0.3%/千字 |
| L2 擦边暗示 | 腰软 / 眼神勾魂 / 夜里不老实 | V3 + 激进档 | 方案简介 1-2 处 |
| L3 禁止红线 | 具体生殖器官词 / 具体性行为 / 辱骂族群 | 全禁 | 0 |

### 6.5 方案输出模板（含创意指纹）

```markdown
## 方案 N：【书名】

**题材**：XX
**主角**：XX（一句话背景）
**核心冲突**：XX
**金手指**：XX（一句话爆点）
**前三章钩子**：XX

🧬 创意指纹
- 命中元规则：M03 信息不对称 / M07 欲望悖论
- 扰动对：外卖员 × 阎王殿 / 直播 × 古代科举
- 金手指维度：GF-1✅ 社交类 / GF-2✅ 代价明确 / GF-3✅ 一句话爆点
- 语言档位：V2 烟火接地气（示例：「阎王他妈的又扣我一天阳寿」）
- 反向规避：避开当日榜单"重生归来"、"都市神医"
```

## 附录 C：`/ink-inspire` 补充命令规格（独立 PRD 实施）

作为 `/ink-init --quick` 的辅助工具，用于用户已有方案但卡在某个环节（主要是金手指），希望单独做脑暴。

### 核心能力

- `/ink-inspire --金手指 10` —— 基于当前已确定题材，一次生成 10 个候选金手指供选
- `/ink-inspire --书名 10` —— 10 个候选书名（含修辞标签）
- `/ink-inspire --人名 10` —— 10 个候选人名（按风格档位）
- `/ink-inspire --冲突 10` —— 10 个核心冲突
- `/ink-inspire --扰动 10` —— 从种子库直接抽 10 对扰动对供用户发散

### 与 quick 的关系

- 复用 Layer 1 元规则 + Layer 2 种子库 + Layer 3 扰动引擎
- 复用金手指三重硬约束
- 不重复实现创意体系，只是换了入口 + 批量输出

### 实施位置

独立 PRD `tasks/prd-ink-inspire.md`，本 PRD 完成后再单独开启实施。
