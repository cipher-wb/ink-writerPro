# 金手指设计模板库

> **目的**: 提供系统化的金手指设计框架，确保金手指设定的合理性、可视化和爽点输出能力。

---

## 📋 金手指设计核心原则

### 1. 功能性原则
- **明确定位**: 金手指必须有清晰的核心功能（升级辅助/战斗增幅/资源获取）
- **成长性**: 金手指应随主角成长而升级，避免"一成不变"
- **限制性**: 必须有合理限制（冷却时间/消耗资源/使用条件），避免"无敌外挂"

### 2. 可视化原则
- **面板化**: 使用面板/属性显示，给读者直观的数据反馈
- **进度条**: 技能熟练度、经验值等使用进度条展示
- **即时反馈**: 使用后立即显示效果（+10 力量/境界突破提示）

### 3. 爽点嵌入原则
- **获得爽**: 金手指首次获得时的仪式感
- **成长爽**: 金手指升级/解锁新功能
- **使用爽**: 金手指在关键时刻发挥作用（打脸/逆袭）

| 字段 | 说明 | 示例 |
|------|------|------|

## 🧩 代价/限制库（v2.0 重构 — 8 大类 × 强制抽样）

> **重要变更**：从 v2.0 起，金手指代价**禁止 LLM 自由发挥**。所有代价必须从 `data/golden-finger-cost-pool.json` (~120 条) 强制抽取 1 主代价 + 2 副代价。本节仅列大类索引，具体条目请查 cost-pool.json。

- **C01 生理代价 Physical**（消耗型/永久型/累积型）：寿命削减 / 气血损耗 / 五感丧失 / 残缺 / 经脉损伤 / 体质异变 / 业障累积 / 黑色印记 / 异力夺舍 等 15 条
- **C02 精神代价 Psychological**：记忆丢失（**high 频趋同区，慎用**）/ 情感剥离 / 精神污染 / 噩梦 / 智商下降 / 心魔 / 性格异化 等 15 条
- **C03 关系代价 Relational**：至亲代偿 / 因果转嫁 / 被遗忘（户籍册擦除/朋友逐一忘记）/ 缘分剥离 / 家族气运 / 友人生隙 等 15 条
- **C04 资源代价 Material**：金钱蒸发 / 账户扣款 / 气运透支 / 功德扣除 / 武器损坏 / 灵石燃烧 / 借贷加倍 等 15 条
- **C05 世界规则代价 Cosmic**：天劫提前 / 雷罚 / 行踪被天道标记 / 高维存在察觉 / 世界排斥度 / 时空错乱 / 因果债 等 15 条
- **C06 限制规则代价 Mechanical**：冷却（**high 频趋同区**）/ 触发苛刻 / 随机性 / 反向报应 / 等价祭品 / 杀人献祭 / 道德绑架 等 15 条
- **C07 身份社会代价 Sociopolitical**：身份暴露 / 系统标记 / 名声妖魔化 / 阵营变更 / 师门驱逐 / 资格剥夺 / 同行围猎 等 15 条
- **C08 生态媒介代价 Environmental**：触发即下雨/雷鸣 / 植物反应 / 鸟兽噤声 / 手机信号干扰 / 镜头无焦 / 钟表停摆 / 影子异常 等 15 条

### 抽样配方

| 项 | 数量 | 约束 |
|----|------|------|
| 主代价 main_cost | 1 条 | `trope_freq ∈ {low, rare}`，`rarity ≥ 3`，与金手指 dimension 不同源 |
| 副代价 side_costs | 2 条 | 至少 1 条跨大类；`high` 配额上限 1/3 |
| 阶梯轴 tier_target | 4 档 | `T1` 小用 → `T2` 中用 → `T3` 大用 → `T4` 极限，必须对应章节阶段 |

### 高频趋同区（黑名单）

下列代价在网文/影视语料中已严重磨损，作为**唯一主代价**直接 GF2=0：

- C02-MEM-016 记忆丢失（千与千寻/记忆传授人/永生美雪等高频意象）
- C01-CON-001 寿命削减（经典扣寿）
- C04-RES-057 灵石燃烧
- C06-CD-076/077 冷却时间

**允许做法**：作为**副代价**配合非趋同主代价使用（占 1/3 配额内）。

### 阶梯轴 T1-T4 演化模板

每个金手指必须在 `escalation_ladder` 字段给出三阶段演化：

```json
{
  "escalation_ladder": {
    "ch1":       {"main_cost_tier": "T1", "scene_anchor": "首章具体场景描写≥30字"},
    "ch10":      {"main_cost_tier": "T2", "scene_anchor": "第10章升级后场景描写≥30字"},
    "late_game": {"main_cost_tier": "T3 或 T4", "scene_anchor": "卷末或全书高潮场景描写≥30字"}
  }
}
```

**阶梯递增不可逆**：`ch1.tier ≤ ch10.tier ≤ late_game.tier`，禁止 ch10 比 ch1 还轻（系统会以 GF5=0 拒绝）。

## 🛡️ 反制方式库（常见）

- **规则屏蔽**：敌对规则让金手指失效  
- **断供**：资源/系统点数被切断  
- **镜像对抗**：反派拥有同类能力  
- **误导利用**：系统提示被反向利用  
- **代价转移**：反派让主角承担更多代价  

---

## ⏱️ 爽点/反馈节奏模板

- **小回报**：每 2-3 章至少 1 次可感知反馈  
- **中回报**：每 8-10 章一次明显升级  
- **大回报**：每 20-30 章一次版本更迭  
- **代价兑现**：每次大回报前后必须兑现代价

### 类型 2: 随身空间流

**核心机制**: 私密储物空间 + 特殊功能

**标准空间属性**:

```markdown
【随身空间】

基础参数:
- 空间大小: 100 平方米 (可升级)
- 时间流速: 外界 1 天 = 空间 10 天
- 种植加速: 灵药成长速度 +500%
- 储物功能: 无限叠加同类物品

特殊功能:
1. [时间静止] - 食物/药品放入后保持新鲜
2. [灵气浓郁] - 空间内修炼速度 +300%
3. [意念收取] - 接触物品即可收入空间
4. [活物收纳] - 可收纳灵兽/战俘(需消耗灵力)
```

**升级路径**:
```markdown
等级 1 → 100 m² (初始)
等级 2 → 500 m² (需筑基期 + 1000 灵石)
等级 3 → 1000 m² (需金丹期 + 10000 灵石)
等级 4 → 小世界 (需元婴期 + 特殊机缘)
```

**爽点设计**:
- **囤货爽**: 末世/战乱前疯狂囤积物资
- **种植爽**: 空间种植灵药，快速获取资源
- **隐藏爽**: 关键时刻从空间掏出大杀器

### 无金手指（纯成长路线）

**适用场景**: 言情、现实向、知乎短篇或反套路题材。

**替代增强点**:
- 主角天赋：专长/天赋/极强意志

**注意**: 无金手指不等于“无爽点”，需把爽点转移到人设、选择、代价与逆袭上。

## 🛠️ 金手指设计工作流

### Step 1: 确定核心定位

**问题清单**:
1. 主角最大的短板是什么？（修炼速度慢/资源缺乏/信息不足）
2. 金手指要解决什么问题？
3. 金手指的独特卖点是什么？

### Step 2: 设定获得方式

**常见方式**:
- ✅ 意外融合（捡到古戒/被雷劈）
- ✅ 传承激活（血脉觉醒/祖传宝物）
- ✅ 系统降临（神秘声音/光幕弹出）
- ❌ 避免：凭空出现，无任何理由

### Step 3: 设计面板/界面

使用 Markdown 表格或 ASCII 艺术设计可视化面板

### Step 4: 规划升级路线

```markdown
等级 1 → 等级 2 → 等级 3 → ... → 最终形态
(对应主角境界: 练气 → 筑基 → 金丹 → ...)
```

### Step 5: 嵌入爽点节点

- 首次获得: 震撼感
- 首次使用: 立竿见影的效果
- 版本升级: 解锁新功能
- 关键时刻: 金手指救命/逆袭
