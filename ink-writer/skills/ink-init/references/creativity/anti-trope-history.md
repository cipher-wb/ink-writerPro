# 反套路库：历史/穿越/古代/年代

> 覆盖题材：历史古代、历史脑洞、抗战谍战、年代、历史穿越
> 用途：在 `/ink-init` Step 5 创意约束包生成时加载，主动预警套路陷阱

## 高危套路清单

### S级套路（极度泛滥，强烈建议避免）

| 套路 | 问题 | 反套路方向 |
|------|------|----------|
| **现代人碾压古代人** | 穿越者自带全知视角，古人沦为NPC | 穿越者的现代知识在古代有严重水土不服，需要适应和妥协 |
| **全员跪舔主角** | 主角随便发明个东西就震惊朝野 | 古代有自己的智慧体系，主角的"发明"可能已被本地人用不同方式解决 |
| **历史人物工具化** | 真实历史人物沦为主角升级道具 | 历史人物有自己的立场和利益，与主角合作时有真实博弈 |
| **种田万能公式** | 炒菜→赚钱→买地→称霸，无阻力递进 | 种田有季节/气候/疫病/地方势力等真实阻力，每一步都有代价 |
| **抗日神剧化** | 敌人智商为零，主角以一敌百 | 敌人有组织有纪律有资源优势，胜利来自策略/牺牲/情报而非蛮力 |

### A级套路（常见但可改良）

| 套路 | 改良方向 |
|------|---------|
| **先知先觉炒股/投资** | 主角知道大趋势但不知细节，执行中遇到信息时差 |
| **一口流利古文** | 语言障碍是真实问题，方言/文白差异/身份暴露风险 |
| **后宫权谋万能** | 权谋有信息成本和信任消耗，过度算计会失去真诚盟友 |
| **科技树狂点** | 科技需要产业链和原料基础，不是有知识就能造出来 |
| **改变历史无后果** | 蝴蝶效应是真实的，改变一件事可能破坏其他有利条件 |

## 历史题材爽点替代方案

| 传统爽点 | 替代方案 |
|---------|---------|
| 现代知识碾压 | 融合古今智慧的创新解法（古人贡献关键本地知识） |
| 预知历史事件 | 知道结果但不知过程，过程中的随机性制造紧张感 |
| 发明现代物品 | 用古代材料和工艺"近似实现"，有瑕疵但够用 |
| 朝堂一言定鼎 | 多方博弈中的关键一票，代价是欠下其他方的政治债 |
| 战场以少胜多 | 局部战术胜利+战略层面的代价交换 |

## 历史真实感检查清单

- [ ] 主角的优势有明确来源且有使用代价吗？
- [ ] 古代角色有独立思考能力和自主行动吗？
- [ ] 历史事件的改变有合理的蝴蝶效应吗？
- [ ] 社会制度/阶级/性别约束有真实反映吗？
- [ ] 物质条件限制（交通/通信/医疗）有体现吗？
