# ink-writer 命令参考表 (v9.0)

## 主入口（日常使用）

| 命令 | 说明 | 使用频率 |
|------|------|---------|
| `/ink-init` | 交互式创建小说项目（书名/题材/角色/世界观/力量体系） | 项目创建时一次 |
| `/ink-auto N` | **全自动写 N 章**——写作+审查+修复+规划+报告，一条命令搞定一切 | 每天 |
| `/ink-resume` | 从中断点恢复写作（自动检测断点位置） | 中断后 |
| `/ink-migrate` | 将旧版项目（v8.x）迁移到 v9.0 架构（三阶段：发现→迁移→审计） | 升级时一次 |

> **90% 的使用场景只需要 `/ink-auto N`。**

## 高级命令（调试/精细控制）

以下命令在 `ink-auto` 内部自动调用，通常不需要手动执行。

### 手动写作

| 命令 | 说明 | 典型场景 |
|------|------|---------|
| `/ink-write` | 写一章（完整 9 步流水线，2200 字起步） | 需要逐章精细控制时 |
| `/ink-plan N` | 生成第 N 卷详细大纲（节拍表+时间线+章纲） | 手动调整大纲时 |

### 手动审查

| 命令 | 说明 | 典型场景 |
|------|------|---------|
| `/ink-review [范围]` | 10 Agent 多维审查（Core/Full/Full+ 三档） | 手动审查特定章节 |
| `/ink-audit` | 数据一致性审计（state.json↔index.db 同步检查） | 怀疑数据错乱时 |
| `/ink-macro-review` | 宏观结构审查（支线健康/角色弧光/承诺审计） | 每卷结束时 |

### 数据维护

| 命令 | 说明 | 典型场景 |
|------|------|---------|
| `/ink-resolve` | 处理 AI 无法确定的实体消歧（唯一需要人工的操作） | 积压 >20 条时 |
| `/ink-query` | 查询项目状态（角色/伏笔/金手指/势力） | 想了解当前状态时 |
| `/ink-learn` | 从会话和审查数据中提取写作模式 | 写完一批后总结经验 |
| `/ink-dashboard` | 启动可视化 Web 面板（只读） | 想看全局视图时 |

## 弃用命令

| 命令 | 替代 | 说明 |
|------|------|------|
| `/ink-5` | `/ink-auto 5` | ink-auto 提供更智能的检查点和自动修复 |
