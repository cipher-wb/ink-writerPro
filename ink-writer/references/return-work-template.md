# 返工清单模板

> 本模板定义了 `/ink-review` Step 7 输出的"返工清单"标准格式。
> 审查器发现 critical/high 问题时，按此格式生成清单供用户/Agent修复。

## 格式规范

### 清单头部

```
## 返工清单 — 第 {章节号} 章

- **审查时间**: {ISO时间}
- **审查深度**: {core|full}
- **问题统计**: critical={N} / high={N} / medium={N} / low={N}
- **总体评分**: {score}/100
```

### 问题条目格式

每个问题按以下格式列出，按严重度降序排列：

```
### [{序号}] {严重度} — {来源Checker}

- **问题ID**: {HARD-001 / POWER_CONFLICT / TIMELINE_ERROR / ...}
- **定位**: 第 {X} 段，约第 {Y}-{Z} 行
- **问题描述**: {一句话描述问题}
- **上下文引用**: > "{引用原文片段，不超过50字}"
- **最小修复动作**: {具体的修改建议，可操作}
- **修复选项**（若有多种方案）:
  - 选项 A: {方案描述}
  - 选项 B: {方案描述}
- **注意事项**: {修复时需要注意的连锁影响}
- **验证方式**: 重新调用 {checker名} 确认
```

### 示例

```
### [1] 🔴 critical — consistency-checker

- **问题ID**: POWER_CONFLICT
- **定位**: 第 8 段，约第 45-52 行
- **问题描述**: 主角筑基三层使用了金丹期技能"破空斩"
- **上下文引用**: > "萧炎双手结印，破空斩的剑气撕裂虚空——"
- **最小修复动作**: 将"破空斩"替换为筑基期可用的"破岩斩"
- **修复选项**:
  - 选项 A: 直接替换技能名（最快，保持效果相似）
  - 选项 B: 补充"传承秘法加成"的铺垫（需在前文插入1-2句铺垫）
- **注意事项**: 若选 B，需确保前文有"获得传承"的线索，否则构成新的设定冲突
- **验证方式**: 重新调用 consistency-checker 确认战力一致性
```

## 末尾摘要

返工清单最后附上修复优先级建议：

```
---
**修复优先级建议**：
1. 先修复所有 critical 问题（必须）
2. 再处理 high 问题（强烈建议）
3. medium/low 可择优处理或记录 deviation
4. 修复完成后建议重新运行 `/ink-review` 验证
```
