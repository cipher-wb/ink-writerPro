# Ralph Progress ‚Äî editor-wisdom-fix

## Codebase Patterns
- `run_review_gate` lives in `ink_writer/editor_wisdom/review_gate.py`; the python orchestration entry is `ink-writer/scripts/step3_harness_gate.py`
- Tests live in `tests/editor_wisdom/`; use `python3 -m pytest tests/editor_wisdom/ -v` to run
- `EditorWisdomConfig` from `ink_writer/editor_wisdom/config.py` is the central config; mock with `patch("ink_writer.editor_wisdom.config.load_config")`
- mypy is not installed; use `ast.parse` for syntax validation
- Coverage fail-under=70% is project-wide; individual test runs will show FAIL but tests themselves pass
- Anthropic model IDs are centralized in `ink_writer/editor_wisdom/models.py` (HAIKU_MODEL, SONNET_MODEL); import from there, don't hardcode
- `applies_to` is restricted to enum `['all_chapters', 'golden_three', 'opening_only']`; LLM free-text values are filtered in `05_extract_rules.py`
- `GOLDEN_THREE_CATEGORIES` is defined in `golden_three.py` and also mirrored in `05_extract_rules.py`; consumer-side filtering uses category, not applies_to
- `Rule` dataclass has a `score: float = 0.0` field; always populated by `Retriever.retrieve`
- FAISS index is `IndexFlatIP`; use `index.reconstruct(i)` to extract individual vectors for category-filtered search
- `Rule` dataclass requires `why: str` field (no default); tests must include it: `Rule(id=..., rule=..., category=..., why="...", severity=...)`
- `cmd_rebuild(from_step=N)` supports resuming from step N (1..6); skips earlier steps
- `02_clean.py` uses `hashlib.blake2b` for minhash signatures (deterministic, not `hash()`)
- `context_injection.py` queries ALL 4 `GOLDEN_THREE_CATEGORIES` for ch<=3, not just 'opening'
