# Ralph Progress — narrative-coherence

## Codebase Patterns
- Agent specs: ink-writer/agents/*.md (21+ files)
- Main workflow: ink-writer/skills/ink-write/SKILL.md (~2000 lines)
- Index manager: ink-writer/scripts/data_modules/index_manager.py (SQLite schema + CRUD)
- Data-agent: ink-writer/agents/data-agent.md (Steps A-F with substeps B.1-B.13)
- Context-agent: ink-writer/agents/context-agent.md (boards 1-14 + 8.5/8.7/8.8/11-16, 8.7 has 📌/⚠️/🔴 three-tier warnings)
- Writer-agent: ink-writer/agents/writer-agent.md (iron laws L1-L6, MCC self-check 7 items)
- Outline-compliance-checker: ink-writer/agents/outline-compliance-checker.md (O1-O6)
- Logic-checker: ink-writer/agents/logic-checker.md (L1-L8)
- chapter_memory_cards table stores per-chapter metadata including summaries
- protagonist_knowledge table tracks what protagonist knows per entity
- relationship_events table tracks relationship changes with polarity/strength
- Tests: tests/ directory + ink-writer/scripts/data_modules/tests/, pytest
- SKILL.md flow: Step 1→2A→2A.1→2A.5→2B→2C→3→4→4.5→5→5.5→6
- Writer-agent post-write chain: MCC自检 → 自洽回扫(SC-1~SC-4) → 输出正文。Both persist to .ink/tmp/
- Schema versioning: SCHEMA_VERSION constant in index_manager.py

- chapter_memory_cards table: scene_exit_snapshot column pre-added via _ensure_column (for US-003)
- Negative constraints methods are on IndexManager class directly (not in a mixin)
- Adding a new board to context-agent.md requires 7 touch points: output overview, budget, truncation priority, degradation table, board definition, success criteria, Step 5.5 preserved fields
