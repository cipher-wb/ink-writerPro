# Ralph Progress — logic-fortress

## Codebase Patterns
- Agent spec files are markdown in ink-writer/agents/ (e.g. writer-agent.md, context-agent.md, consistency-checker.md)
- Main workflow spec: ink-writer/skills/ink-write/SKILL.md
- Review gate spec: ink-writer/skills/ink-write/references/step-3-review-gate.md
- Polish guide: ink-writer/skills/ink-write/references/polish-guide.md
- Core constraints: ink-writer/references/shared/core-constraints.md
- All agent specs use markdown with YAML-like structure (sections, subsections, bullet lists)
- Existing checker pattern: frontmatter → purpose → check layers → severity definitions → MUST_NOT_PASS → output format → examples
- 新建checker必须引用3个模板: checker-output-reference.md, checker-input-rules.md, checker-load-context.md
- checker JSON输出的私有扩展字段（如logic-checker的layer字段）放在issues[]内，不替代统一schema字段
