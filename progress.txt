- **Learnings for future iterations:**
  - **LLM 返回 char_range 与 text 长度不一致是常态**：LLM 可能返 `text_len < char_range[1]-char_range[0]`（截断）或相反。非超长情况下**保留 LLM 原始 char_range**（信任 LLM 的源定位），只有超长切分时才按 sub.len 累加 sub_start。错误做法是一律从 text_len 重算，会让 happy path 测试失败（char_range[1] 被改成 len(text)）。
  - **`re.search(r"\{.*\}", text, re.DOTALL)` 是 LLM JSON 恢复的"双保险"**：许多 LLM（含 Haiku）偶尔会在 JSON 外包 ```json ... ``` 或前置解释文字。先 `json.loads(raw)` 快路径；失败再正则抓第一个 `{...}`；都失败才 None 进入重试。
  - **`_rechunk_oversize` 的句末边界阈值 `last_period > 100`**：保证切分点不过于靠近起始（否则第一块可能 < 200 违反 min_chunk_chars 语义）。如果最后一段（`cursor + last_period + 1 >= len(text)`）也要走 `cut=end` 避免把剩余拼成长尾。
  - **Haiku 入口不需要 `__main__`**：scene_segmenter.py 纯 library，无 CLI 入口，因此不需要 `enable_windows_utf8_stdio()` 调用；audit 红线只扫含 `if __name__ == "__main__":` 的 .py。

- **Learnings for future iterations:**
  - **CLI 测试核心断言不是"跑通"而是"未触发某分支"**：`build_qdrant = MagicMock(side_effect=AssertionError(...))` 让 side_effect 抛错来保证 dry-run 路径**根本不该调** — 比事后 `assert not called` 语义更强（即便覆盖率工具误报，side_effect 也会让错误路径立刻炸出 stack trace）。
  - **`DEFAULT_DATA_DIR` 必须放 module 级别以便 test monkeypatch**：如果内嵌在 _cmd_ingest 函数里则 test 无法干预输出路径，会污染真实 `data/corpus_chunks/` 目录（非 idempotent 副作用）。遵循的通用模式：**所有"默认相对路径"常量都放 module top-level**，方便 tests monkeypatch 到 tmp_path。
  - **`datetime.now(UTC)` 替代 `datetime.utcnow()`**：Python 3.12+ 弃用 `datetime.utcnow()`，ruff DTZ003 会警告。改用 `from datetime import UTC, datetime` + `datetime.now(UTC).strftime("%Y-%m-%d")`。
  - **argparse subparsers `required=True` 默认不弹**：Python 3.7+ subparsers 缺省 `required=False`，必须显式 `sub = parser.add_subparsers(dest="command", required=True)` 才会在 `python -m x.cli`（无子命令）时 rc=2 报错。
  - **rebuild/watch 先留 stub（返 rc=2 + stderr 提示）再分别由 US-006/US-007 补实现**：比一次性实现再测试路径耦合风险低。stub 返 2 而非 0 便于 CLI 级 smoke test 捕获遗漏。
  - **`_already_indexed` 以 chunks_raw.jsonl 作为 resume 游标而非 metadata.jsonl**：chunks_raw 在 segmenter 成功后立即写入（tag/index 失败不影响），resume 时不应因 tag/index 失败而反复跑 segmenter（调 LLM 贵）。原则：**resume 游标应放在最便宜工序的输出之后**，防止重试的 cost 放大。
  - **`_build_*_client` helper 存在的意义就是测试用 monkeypatch**：不要把 `import anthropic; client = anthropic.Anthropic()` 直接写进 `_cmd_ingest`（这样 test 只能走 sys.modules 替换，脆）。helper 函数让 test 可以一行 `monkeypatch.setattr(cli_mod, "_build_anthropic_client", ...)` 精确隔离。
