## Codebase Patterns
- YAML loaders 模式：参考 `ink_writer/reader_pull/config.py`，用 `Path(__file__).resolve().parent.parent.parent / "config" / "<name>.yaml"` 定位默认路径
- `open()` 全部带 `encoding="utf-8"`（CLAUDE.md Windows 兼容守则强制）
- ruff isort 格式：`import pytest` 与第一方 import 之间不留空行（同段视为一组）
- pytest.ini testpaths 单行追加新目录；不重复添加已有目录
- 项目无 mypy/pyright；typecheck 等价于 `ruff check` 通过
- `python3` 而非 `python`（macOS 默认 alias）
- ruff 默认开 UP017：`from datetime import timezone; timezone.utc` 必须改为 `from datetime import UTC; UTC`（Py 3.11+ alias）
- M3 期 evidence_chain 写盘：默认 base_dir 是 `Path("data")`；测试传 tmp_path 隔离
- LLM client stub 标准模式：`FakeLLMClient(responders=[...])` + `.messages.create(**kwargs).content[0].text`；FIFO 队列，超出列表时复用最后一个；调用 kwargs 记到 `.calls` 便于断言 prompt 内容（参考 `tests/writer_self_check/conftest.py`）
- prompt 模板中 JSON schema 示例的 `{` `}` 必须写成 `{{` `}}`，否则 `.format()` 会爆 KeyError
- M4 期 LLMClient 接口：复用 `scripts/corpus_chunking/llm_client.LLMClient`；checker 调用形如 `llm_client.complete(prompt=..., model="glm-4.6", max_tokens=..., temperature=0)`，返回 raw str；checker 内自行 `_extract_json` 去 markdown 包裹
- M4 checker 文件结构：`ink_writer/checkers/<name>/{__init__.py, models.py, checker.py, prompts/check.txt}`；agent.md 在 `ink-writer/agents/<name>-checker.md`；测试在 `tests/checkers/<name>/{__init__.py, test_checker.py}`（与 M3 conflict_skeleton / protagonist_agency 同源）
- M4 evidence_chain 派生：复用 `ink_writer/evidence_chain/models.EvidenceChain`，加 `phase: str = "writing"` 字段；planning 写盘走 `planning_writer.write_planning_evidence_chain`，路径是 `<base_dir>/<book>/planning_evidence_chain.json`（与 chapter `<book>/chapters/<ch>.evidence.json` 不同目录）；多次写入合并 `stages` 列表
- M5 case schema 扩展：`Case` 加 3 个 optional 字段（`recurrence_history: list[dict]` / `meta_rule_id: str | None` / `sovereign: bool`）；`from_dict` 容错读取（缺字段 fallback 默认值），保证现有 410 个 case yaml 不强制 backfill
- M5 Layer 4 复发追踪：仅同 book 内复发触发；severity 升级 P3→P2→P1→P0；P0 已顶则靠 `recurrence_history` list 长度体现 recurrence_count
- M5 Layer 5 元规则浮现：N=5 + LLM similarity > 0.80；先按 tag 重叠粗分组（_candidate_clusters_by_tags）避免 N² LLM 调用；跳过 sovereign + 已 meta_rule_id 的 case
- M5 dashboard aggregator 4 大指标的"M5 占位"：repair_speed_days / checker_accuracy 是 placeholder 7.0/0.0（真实数据需 case 加 resolved_at + ink-learn 抽样工作流，M5 之外）
- M5 evidence_chain 加 `channel: str | None = None` 字段（A/B 通道）；planning_writer 透传到 stages dict；现有 evidence_chain 文件无 channel 字段 fallback None（向后兼容）
