  - **PyYAML 解析 replacement 字段嵌内中文直引号会炸**：`replacement: 具体情绪词（如"一阵心慌"）` 里的 `"` 会被 YAML 解释为双引号串，吃掉半行；必须给整个 value 单引号包裹 `'...'`。一次性脚本扫描所有 `replacement:`/`word:` 行，遇 `"` 就单引号包装、内部 `'` 转义为 `''`，是最稳的 fix。写 YAML 文档含引号字符时，源生成阶段就该统一单引号，不要依赖多行块标量。

  - **loader 的 yaml import 延迟**：`load_blacklist` 内 `try: import yaml` 而非顶部 import——测试里 `pytest.importorskip("yaml")` 挡掉无 PyYAML 环境；模块层 import `yaml` 后者测试里 `monkeypatch.setattr(yaml_mod, "safe_load", spy)` 才能精准 hook 到本次 load 的调用栈。
  - **`Path.exists() and is_file()` 双判**：仅 `exists()` 会放过符号链接指向的目录；blacklist 设计上是单文件资产，双判更防御。
  - **公开 API 跟随 `__init__.py`**：把 `load_blacklist` / `Blacklist` / `BlacklistEntry` / `clear_cache` 都 re-export——但 `CATEGORIES` 常量留在 `blacklist_loader` 子模块（单元测试才需要），避免污染默认 namespace。
