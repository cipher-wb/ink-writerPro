## Codebase Patterns
(承接自前轮 ralph runs，保留跨 PRD 通用条目。逐轮新增追加到下方。)

- **UTF-8 编码硬约束**：所有 `open()` / `Path.read_text()` / `Path.write_text()` 必须带 `encoding="utf-8"`。scripts/fix_utf8_encoding.py 可批量修复；scripts/audit_cross_platform.py 是 9 类风险单一扫描入口。
- **Windows 兼容守则**：新增 .sh 必须配 .ps1（UTF-8 BOM）+ .cmd 双击包装；新增带 `if __name__ == "__main__":` 的脚本在 main 开头调用 `runtime_compat.enable_windows_utf8_stdio()`（Mac no-op）。详见 CLAUDE.md。
- **agent 规格统一目录**：所有 agent 规格在 `ink-writer/agents/`（非 `ink_writer/`，US-402 已消除双目录）。
- **pytest.ini testpaths 必须显式注册**：新建 `tests/<module>/` 目录后必须追加到 `pytest.ini` 的 `testpaths` 行，否则 pytest 不收集；项目覆盖率门禁 `--cov-fail-under=70`。运行时可用 `pytest tests/<new-dir>` 直接目标调用绕过 testpaths，适合 pre-US-002 的单任务验证。
- **Python 包 vs 插件目录命名**：`ink_writer/` 是 Python 包（pyproject.toml 注册），`ink-writer/` 是 Claude Code 插件（agents / skills / scripts）；不可混用。
- **Ruff 是 lint gate**（`python3 -m ruff check`）。常见自动修复：UP009（去 `# -*- coding: utf-8 -*-`）、UP035（`typing.Sequence` → `collections.abc.Sequence`）、I001（import ordering）。新增 .py 提交前先跑 `ruff check --fix <paths>`。
- **无 mypy/pyright 配置** — PRD "Typecheck passes" 的实际含义：`python3 -m py_compile <file>` + `ruff check` 同时干净。
- **断链检测标准写法**：`p.is_symlink() and not p.exists()`（`Path.exists()` 会 follow 符号链接，所以返 False 即 target 丢失）。
- **仓库级 audit 红线会扫全仓 .py**：`tests/audit/test_cli_entries_utf8_stdio.py::test_repo_has_no_cli_entry_missing_utf8_stdio`（任何含 `__main__` 的 .py 必须出现 `enable_windows_utf8_stdio` 字面量）与 `tests/core/test_safe_symlink.py::test_repo_has_no_bare_symlink_calls_outside_primitives`（禁止 `os.symlink` / `Path.symlink_to` 裸调用，须走 `runtime_compat.safe_symlink`；测试 fixture 处用 `# c5-ok: 原因` 豁免）。**新增任何 .py 之后一定要跑这两个测试**，不能只依赖子目录 pytest 绿灯。
- **PEP 440 `~=X.Y` 语义**：`>=X.Y, <(X+1).0`（允许次版本号上升到 `(X+1).0` 之前），不是"只允许 X.Y.z"。要锁次版本号写 `~=X.Y.0`。PRD 里的"输出 1.12.x"类措辞是估值，判合规看约束不看措辞。
