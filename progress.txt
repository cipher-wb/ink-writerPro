## Codebase Patterns
- Schema 修改严格保持 `additionalProperties: false`：任何新顶层字段必须在 properties 显式声明，否则现存数据立即拒收。
- case_id pattern 容纳多前缀的方式：在主 alternation 段加分支（如 `LR(-[0-9]{4})?`），保留 `(M[0-9]+-)?` 与末尾 `[0-9]{4}$` 不动；每加一个前缀都要回归 `tests/case_library/test_schema_backward_compat.py::test_all_existing_cases_still_validate` 全 410 份。
- pytest.ini `testpaths` 是单行长串 — 新增目录追加在末尾即可，不要重排已有目录避免 diff 噪音；行后注释保持原样。

- ID 分配复用 `case_library._id_alloc.allocate_case_id(cases_dir, prefix=...)`：传不同 prefix 自动隔离 counter file（`.id_alloc_<sanitized_prefix>.cnt`）+ glob 兜底防外部手工添加 yaml 让 counter 落后；并发安全（独占文件锁，4 worker spawn 0001..0004 无空洞已验证）。新 case 类型不需要再写 ID 分配器，写 `live_review.case_id.allocate_live_review_id` 这种薄封装即可。

- **`statistics.quantiles(method='exclusive')` 在小样本上的精确数值**：n=4 quantiles 用 m_k=k*(N+1)/4 位置插值。N=7 [50,55,60,65,70,75,80] → [55,65,75]；N=5 [60,62,65,70,75] → [61,65,72.5]；N=5 [60,65,70,75,80] → [62.5,70,77.5]。需要 N≥2 否则抛 StatisticsError；N=1 用 `_safe_quantiles` 兜底返 (None, float(scores[0]), None)。聚合脚本对全 None score 的 genre 直接返 score_mean=None / score_median=None / score_p25=None / score_p75=None — 这样既符合 schema "number|null" 也不混淆"统计量为 0"与"无数据"。

- **30 份测试 fixture 用一次性 generator 生成后删除**：写 `/tmp/gen_lr_fixtures.py` 一次性脚本生成 30 份 yaml → 校验 → 删脚本；30 份 yaml 作为静态 fixture 提交，构建 reproducibility 由 PR diff + schema 校验回归共同保证。这套模式比 conftest.py 动态创建 fixture 更稳（不依赖 generator 长期存在被误改），下次扩 fixture 数量复用此模式。
- **common_complaints 严格降序需 fixture 设计配合**：测试用 `>=` 验证非严格降序就够；想测严格 `>` 必须从 fixture 端保证唯一频率。我设计都市 negatives 为 opening=5 / pacing=4 / golden_finger=3 / hook=2 / character=1 五档严格不同 → 满足 "频率严格降序" AC。如果 PRD AC 说"非严格"则 fixture 可以放任 ties。
- **subprocess fixture 嵌套 pytest fixture（@pytest.fixture aggregate(out_path)）**：`aggregate` fixture 调用 `_run` + 解析 json，所有依赖它的测试都共享一次 subprocess 调用 — 13 用例只跑一次脚本，比每用例独立跑省时。但需注意 fixture scope 默认 function，要 module-level 共享得显式 `scope="module"` + tmp_path_factory 替代 tmp_path。
- **`statistics.median` 的偶数样本返回中位数算术平均**：N=2 返 (a+b)/2；statistics.quantiles 在 N=2 用 m_k=k*3/4 也插值。聚合脚本不需要特殊处理偶数样本，直接调即可。但要警惕 N=1 时 quantiles 抛错 — `_safe_quantiles` 必须兜底。
- **`yaml.safe_load` 对 None / 空文件返 None**：load_cases 必须 `if data is None: continue` 防御空 yaml；30 份 fixture 不会触发但生产环境若有人手工删空内容会暴露。所有 `yaml.safe_load` 后建议加这道防线。
