- **Mac 字节级一致承诺**（v19 起）：所有 Windows 特化代码走 `if sys.platform == "win32":` 分支，`.sh` 文件字节不变——本轮修复须严格保留。
- **COMPLETE 信号锚定**（v21 US-011 起）：`scripts/ralph/ralph.sh` 的 COMPLETE 检测用 `tail -n 50 | grep -qE '^[[:space:]]*<promise>COMPLETE</promise>[[:space:]]*$'`——讨论里引用这个字符串不会触发早停。本轮只需按正常方式在最终总结里输出 `<promise>COMPLETE</promise>` 独占一行。
- **jieba posseg 延迟导入**（US-001 起）：scripts 中的 jieba.posseg 加载 ~200ms；用 `_lazy_jieba()` helper 放到函数体而非顶部 import。模块导入期不触发，避免 pytest 收集期加重。
- **中文对话 unicode 显式写**（US-001 起）：源码里"「」""" 这些弯引号字符容易被编辑器/终端折叠成直引号，导致运行时检测失败。统一用 `"\u201c"`/`"\u201d"` 等显式 unicode 转义。
- **ruff E402 规避**：scripts/ 入口在 `if sys.platform == "win32":` runtime_compat setup 之后才能 import 其他模块，统一加 `# noqa: E402` 到那批 late imports，`from __future__ import annotations` 后置不变。
- **YAML 字符串含 `"` 必须单引号包**（US-003 起）：`replacement: 具体情绪（如"一阵心慌"）` 里的直引号会被 YAML 解释成双引号串，吃掉半行触发 `expected <block end>, but found '<scalar>'`。解决方案：写资产 YAML 时对含任何 `"` 的 scalar value 统一单引号包裹（`'...'`），内部 `'` 转义为 `''`；禁止依赖裸值。已通过 `ink-writer/assets/prose-blacklist.yaml` 验证。

- **Learnings for future iterations:**
  - **PyYAML 解析 replacement 字段嵌内中文直引号会炸**：`replacement: 具体情绪词（如"一阵心慌"）` 里的 `"` 会被 YAML 解释为双引号串，吃掉半行；必须给整个 value 单引号包裹 `'...'`。一次性脚本扫描所有 `replacement:`/`word:` 行，遇 `"` 就单引号包装、内部 `'` 转义为 `''`，是最稳的 fix。写 YAML 文档含引号字符时，源生成阶段就该统一单引号，不要依赖多行块标量。
  - **`…` 软通配符匹配要 min(counts) 防炸**：`宛如…一般` 拆 `["宛如","一般"]` 后对每段独立 count，取 min 作为命中数；不能用 max 或 sum，否则单段高频词（如"一般"）会把命中量拉飞。测试里 `_make([("宛如…一般",...)])` + `"他宛如一尊石像一般立着"` 必须 =1，`"他宛如一尊石像"` 缺"一般" = 0。
  - **mtime_ns 缓存键要扛同刻度覆盖**：`os.utime(target, ns=(future_ns, future_ns))` 把时间戳往前推 1s 整，避免同一秒内连写两次被缓存。测试用 `+1_000_000_000` 纳秒偏移最稳；真实生产环境作者手动改 YAML，mtime 会自然变化。
  - **loader 的 yaml import 延迟**：`load_blacklist` 内 `try: import yaml` 而非顶部 import——测试里 `pytest.importorskip("yaml")` 挡掉无 PyYAML 环境；模块层 import `yaml` 后者测试里 `monkeypatch.setattr(yaml_mod, "safe_load", spy)` 才能精准 hook 到本次 load 的调用栈。
  - **`Path.exists() and is_file()` 双判**：仅 `exists()` 会放过符号链接指向的目录；blacklist 设计上是单文件资产，双判更防御。
  - **公开 API 跟随 `__init__.py`**：把 `load_blacklist` / `Blacklist` / `BlacklistEntry` / `clear_cache` 都 re-export——但 `CATEGORIES` 常量留在 `blacklist_loader` 子模块（单元测试才需要），避免污染默认 namespace。
  - **US-006 writer-agent prompt 引用前 20 条黑名单**：按 YAML 里的定义顺序取前 20 条即可——YAML 已按编辑最高频→中频排序，abstract_adjectives 开头就是"莫名/无尽/难以言喻/仿佛/似乎"这五个最毒的，足够当反例。
  - **US-008 polish simplification 直接用 `bundle.match(paragraph)` 枚举命中**：替换策略消费 `entry.replacement` 字段里的 showing 提示；真要全自动替换请警惕 replacement 多是启发式描述而非可插入文本——polish prompt 层面应让模型理解"这是指引不是直接替换串"。
