# Ralph Progress — wordcount-and-progress

## Codebase Patterns
- Agent spec files are markdown in ink-writer/agents/ (e.g. writer-agent.md)
- Main workflow spec: ink-writer/skills/ink-write/SKILL.md (~1700 lines)
- Workflow manager: ink-writer/scripts/workflow_manager.py (manages workflow_state.json, has subcommands via argparse)
- Auto batch script: ink-writer/scripts/ink-auto.sh (~1000 lines bash)
- Word count validation is in SKILL.md Step 2A.5 (~L865-880)
- Writer-agent word limits at ~L176 and ~L331
- ink-auto.sh main loop starts ~L1000+, uses [$i/$N] format for chapter counting
- ink-auto.sh 进度条函数：print_chapter_progress() ~L76, print_checkpoint_progress() ~L129, format_duration() ~L63
- ink-auto.sh 检查点编排器 run_checkpoint() 内使用 cp_steps/cp_status 数组追踪子步骤进度
- ink-auto.sh 内层进度解析：parse_progress_output() 函数 ~L536，get_step_name() ~L517
- run_cli_process() 用 parse_progress_output 替代 tee，同时写日志+解析进度+透传普通输出
- workflow_manager.py subcommands: begin-task, complete-step, complete-task, etc.
- [INK-PROGRESS] events are emitted to stdout by start_step(), complete_step(), complete_task() — grep for "[INK-PROGRESS]" to parse
- Tests in tests/ directory, run with pytest
- 进度输出规范在 SKILL.md ~L95，定义 12 步映射和事件格式
- 12步进度映射：Step 0/0.7/0.8/1/2A/2A.5/2B/2C/3/4/5/6
