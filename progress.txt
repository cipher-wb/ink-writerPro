## Codebase Patterns
(承接自前轮 ralph runs，保留跨 PRD 通用条目。逐轮新增追加到下方。)

- **UTF-8 编码硬约束**：所有 `open()` / `Path.read_text()` / `Path.write_text()` 必须带 `encoding="utf-8"`（CLAUDE.md Windows 兼容守则第 1 条）。中文项目在 Windows 上不带参数会炸 UnicodeDecodeError。
- **零回归守卫 baseline**：v20.0.0 发版后 `pytest --no-cov` 全量 baseline = 3021 passed / 23 skipped / 0 failed。本轮每个 US 完成前必须跑全量无新增失败。
- **pytest.ini testpaths 同步**：新增的 `tests/核心模块/子目录/` 必须追加到 `pytest.ini:testpaths`，否则默认收集会漏掉。
- **ProjectConfig 读取约定**：章节正文文件命名/后缀由 `ProjectConfig` 决定；新代码不硬编码命名。
- **ralph runtime 文件不入 commit**：`.claude/.ink-current-project`、`scripts/ralph/.last-branch` 是 runtime 状态；归档副本在 `scripts/ralph/archive/` 下受版本控制。
- **kwarg-only 向后兼容**：给既有 public 函数新增参数时放到 `*` 之后（kwarg-only），避免破坏生产/测试调用方的位置参数假设。
- **runtime_compat 原语集中复用**：`ink-writer/scripts/runtime_compat.py` 已经暴露 5 个 helper（`enable_windows_utf8_stdio` / `normalize_windows_path` / `set_windows_proactor_policy` / `_has_symlink_privilege` / `find_python_launcher`）——新增跨平台 helper 统一放这里，不散落。
- **Mac 字节级一致承诺**（v19 起）：所有 Windows 特化代码走 `if sys.platform == "win32":` 分支，`.sh` 文件字节不变——本轮修复须严格保留。
- **跨平台审计单一入口**（US-001 起）：`scripts/audit_cross_platform.py --root <path> --output <md>` 是 9 类风险（C1~C9）的唯一权威扫描器；US-002~US-010 修复完成后必须重跑此脚本验证 finding 数下降，不能引入新类别。
- **C5 symlink 正则坑点**：检测裸 symlink 用 `(?:\bos\.symlink|\.symlink_to)\b`，**不能在 `\.symlink_to` 前加 `\b`**——`)` → `.` 都是非词符，`\b` 不会匹配（test_c5 已固化此用法）。
- **AST 扫描器统一接收 `Iterable[Path]`**：审计脚本各 scan_cN_xxx 函数对 `.py` 文件采用 `safe_read_text` + `ast.parse`，捕获 `SyntaxError` 跳过，避免单文件阻断全量扫描。
