- 原子写盘统一模式（US-008 终版）：`tempfile.mkstemp(prefix=..., suffix='.tmp', dir=...)` 拿 (fd, tmp_name) 后**必须**用内嵌 try/except 包住 `Path(tmp_name)` 和 `os.fdopen(fd)` 这两步：异常路径手动 `os.close(fd)` + unlink tmp（fdopen 失败时 fd 未被 BufferedWriter 接管，外层 `with` 不会兜底，旧实现会泄漏 fd）。然后 `with fh: write+flush+fsync` → `os.replace(tmp, target)`；外层 except 路径再 `tmp.unlink()` 兜底覆盖 replace 之前的所有失败窗口。OSError 抑制统一用 `contextlib.suppress(OSError)`，避免 SIM105 噪音。复用此模式的两个标准实现：`ink_writer/evidence_chain/writer.py:_atomic_write_text` + `ink_writer/rewrite_loop/human_review.py:_atomic_write_text`（两份完全一致）。
- fail-loud 原则：上游 schema 漂移 / 锁失败 / 未知类型应显式抛错而非 silent pass — 参考 commit 47a6fe5 (Windows lock) 与本批次 US-006 (_serialize_case)。
- `ink_writer/_compat/locking.py:locked_file` 是当前唯一的跨平台独占文件锁原语；US-003 计划在同文件加 `shared_locked_file`（read-only 共享锁）；US-004 复用 `locked_file`。
- `ink_writer/rewrite_loop/dry_run.py:_observation_quality_passed` 的 chapters 路径约定：``base_dir / book / 'chapters'``（直接拼，不再走 ``base.name == 'data'`` 启发式）。是 dry-run 内部独立路径假设，与 ``ink_writer/evidence_chain/dry_run_report.py:aggregate_dry_run_metrics`` 期待 ``base_dir/data/<book>/chapters`` 的约定**不同**。两者公共 API 互不影响。
- ``ink_writer/rewrite_loop/dry_run.py:_delivered_rate(chapters_root)`` helper：直接读 ``*.evidence.json`` 计 delivered/total 返 ``float | None``；空目录或读取全失败返 None。dry_run 内部 quality gate 专用，不导出。
- ``ink_writer/_compat/locking.py:shared_locked_file`` (US-003)：Unix 用 ``fcntl.flock(LOCK_SH)`` 真共享锁，多 reader 并发；Windows ``msvcrt`` 无共享锁概念降级为 ``LK_LOCK`` 独占（reader 也走串行）。任何"读频率高、写频率低、不能容忍 writer 中途状态"的文件读路径优先用此 helper。
- ``read_dry_run_counter`` retry 模式：用 ``shared_locked_file`` 拿到 raw='' 但 ``path.stat().st_size > 0`` 表示命中 writer ``truncate→write`` 极小窗口；重读一次（**最多 1 次**，避免无限循环）。这个 read-empty + size-positive 检测模式可被其他"writer atomic-replace 风格 + reader 共享锁"组合复用。
- counter 文件 "open-or-create + lock-internal init" 标准做法（US-004）：``with locked_file(path, "a+") as fh:`` 后立刻 ``fh.seek(0)`` 读 → 空走初始化 → ``fh.seek(0) + truncate + write`` 写回。'a+' 在 Unix/Windows 都 auto-create；O_APPEND 写到 EOF，truncate=0 后 EOF=0 落 0 位置正确。**禁止**再用"锁外 ``if not exists: write_text(...)`` + 锁内救场" 这种旧模式。
- ``write_human_review_record`` 的 ``blocking_cases`` 序列化全部在 ``open()`` 之前的字典字面量列表推导式中完成（``record = {"blocking_cases": [_serialize_case(c) for c in blocking_cases], ...}``）；任何 _serialize_case 抛错天然 fail-before-write，jsonl 文件不会被创建。这是"序列化先于写盘"模式的清晰例证 — 未来类似 append 风格写盘最好都这样组织：先把所有 entry 序列化成 str（``json.dumps``）再 open+write，避免半截行/部分文件污染。

  - 模块级私有函数 (`_hit_case_ids`) 直接被测试 import 是合理做法 (Python 的下划线只是约定不是私有性) —— 比把测试套用到 `propose_cases_from_failures` 全链路绕一大圈再断言派生结果更省时也更精准。tests/learn/test_auto_case.py 现有结构允许混合两种粒度：`test_hit_case_ids_*` 单元 + `test_proposes_*` / `test_skips_*` / `test_throttled_*` 集成。
  - generator 在 `isinstance(x, Iterable)` 下也是 True，但 `isinstance(x, list)` 下是 False。此处恰好是想要的语义：generator 不像 list 那样可重读、易引发其他下游 bug。如未来真的要支持 generator，应显式 `list(value)` materialize 后再处理。

- **Learnings for future iterations:**
  - `tempfile.mkstemp` 返回 `(fd, name)` 的语义陷阱：fd 是裸 OS fd，不被任何 Python 对象拥有；只有 `os.fdopen(fd)` 成功后才被 BufferedWriter 接管。fdopen 抛错时 `with os.fdopen(...) as fh:` 的 `__exit__` 永不触发——外层兜底不会 close fd。所以"先 mkstemp 后 fdopen"的代码必须把 fdopen 单独放在内嵌 try/except 里，except 路径手动 `os.close(fd)`。这个 fd-ownership 边界容易看走眼。
  - 跨平台 fd 计数：Linux 走 `/proc/self/fd`，macOS 走 `/dev/fd`，两者都返目录列表项数。`pytest.skip` 兜底罕见平台。`psutil.Process().num_fds()` 更可移植但本项目无 psutil 依赖，按 `os.path.isdir` 探测两个标准位置最轻量。
  - `with contextlib.suppress(OSError): ...` 是 ruff SIM105 规则建议的标准替换；语义与 `try: ... except OSError: pass` 完全等价但更声明式，且消 SIM105 lint 噪音。改造旧 `_atomic_write_text` 时可顺手把所有兜底点换上，不必单开 PR。
  - parameterized 测试覆盖"两份内容完全一致的不同模块"的标准做法：`pytest.mark.parametrize` 传 `(module_path, invoke_fn, chapters_subpath)`，invoke 用 module-level helper 函数（保持简洁、可被 spawn 序列化如未来要并发跑）。`os.fdopen` mock 路径要带模块前缀（`{module_path}.os.fdopen`），不能 patch `os.fdopen` 全局以免误中其他测试基础设施。
