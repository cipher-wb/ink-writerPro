# Ralph Progress — deep-review-and-perfection

## Codebase Patterns
- IndexManager auto-creates all DB tables on init; never create tables manually in fixtures — let IndexManager.__init__ handle schema, then INSERT test data
- E402 lint warnings are expected for scripts using sys.path manipulation before imports (project_locator, data_modules)
- DataModulesConfig.from_project_root(path) expects `.ink/` dir with `state.json` under project_root
- Review metrics are stored in review_metrics table with JSON-encoded dimension_scores, severity_counts, review_payload_json
- chapter_reading_power table stores per-chapter hook/micropayoff data; tool_call_stats has limited columns (no duration_ms/tokens_used)
- pytest.ini testpaths: `ink-writer/scripts/data_modules/tests tests/baseline`; pythonpath includes `scripts`
- benchmark/corpus_index.json has 117 books; use `collections` field for ranking, `dir` field for filesystem lookup
- benchmark/config.py has SCENE_TYPE_KEYWORDS / EMOTION_KEYWORDS — reuse for text analysis instead of reinventing
