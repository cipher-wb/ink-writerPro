## Codebase Patterns
(承接自前轮 ralph runs，保留跨 PRD 通用条目。逐轮新增追加到下方。)

- **UTF-8 编码硬约束**：所有 `open()` / `Path.read_text()` / `Path.write_text()` 必须带 `encoding="utf-8"`。scripts/fix_utf8_encoding.py 可批量修复；scripts/audit_cross_platform.py 是 9 类风险单一扫描入口。
- **Windows 兼容守则**：新增 .sh 必须配 .ps1（UTF-8 BOM）+ .cmd 双击包装；新增带 `if __name__ == "__main__":` 的脚本在 main 开头调用 `runtime_compat.enable_windows_utf8_stdio()`（Mac no-op）。详见 CLAUDE.md。
- **agent 规格统一目录**：所有 agent 规格在 `ink-writer/agents/`（非 `ink_writer/`，US-402 已消除双目录）。
- **pytest.ini testpaths 必须显式注册**：新建 `tests/<module>/` 目录后必须追加到 `pytest.ini` 的 `testpaths` 行，否则 pytest 不收集；项目覆盖率门禁 `--cov-fail-under=70`。运行时可用 `pytest tests/<new-dir>` 直接目标调用绕过 testpaths，适合 pre-US-002 的单任务验证。
- **Python 包 vs 插件目录命名**：`ink_writer/` 是 Python 包（pyproject.toml 注册），`ink-writer/` 是 Claude Code 插件（agents / skills / scripts）；不可混用。
- **Ruff 是 lint gate**（`python3 -m ruff check`）。常见自动修复：UP009（去 `# -*- coding: utf-8 -*-`）、UP035（`typing.Sequence` → `collections.abc.Sequence`）、I001（import ordering）。新增 .py 提交前先跑 `ruff check --fix <paths>`。
- **无 mypy/pyright 配置** — PRD "Typecheck passes" 的实际含义：`python3 -m py_compile <file>` + `ruff check` 同时干净。
- **断链检测标准写法**：`p.is_symlink() and not p.exists()`（`Path.exists()` 会 follow 符号链接，所以返 False 即 target 丢失）。
- **仓库级 audit 红线会扫全仓 .py**：`tests/audit/test_cli_entries_utf8_stdio.py::test_repo_has_no_cli_entry_missing_utf8_stdio`（任何含 `__main__` 的 .py 必须出现 `enable_windows_utf8_stdio` 字面量）与 `tests/core/test_safe_symlink.py::test_repo_has_no_bare_symlink_calls_outside_primitives`（禁止 `os.symlink` / `Path.symlink_to` 裸调用，须走 `runtime_compat.safe_symlink`；测试 fixture 处用 `# c5-ok: 原因` 豁免）。**新增任何 .py 之后一定要跑这两个测试**，不能只依赖子目录 pytest 绿灯。
- **PEP 440 `~=X.Y` 语义**：`>=X.Y, <(X+1).0`（允许次版本号上升到 `(X+1).0` 之前），不是"只允许 X.Y.z"。要锁次版本号写 `~=X.Y.0`。PRD 里的"输出 1.12.x"类措辞是估值，判合规看约束不看措辞。

- **Learnings for future iterations:**
  - Pattern discovered: **QdrantClient(":memory:") 会在启动时打印 `UserWarning: Failed to obtain server version. Unable to check client-server compatibility`**——这是 qdrant-client 1.17.1 的已知噪声（in-memory 没有 server 可查版本）。文档建议 `check_compatibility=False` 消声，但 1.12 分支无该参数；1.17.1 虽然支持却在 API 上标 internal。**不要为此加 warnings.filterwarnings**——测试 collection 和 CI 都能忍这一条 warning，过滤反而会掩盖真实的 deprecation。US-012/US-013 直接沿用 conftest 的 `in_memory_client`，看到这条 warning 属正常。
  - Pattern discovered: **进程级单例 + `reset_singleton_for_tests()` 是本仓库的标准范式**。好处：(1) 生产调用不必在每层传 client；(2) 测试用 monkeypatch 配合 reset 可切换 config。新写 singleton 时最好显式暴露 reset——否则不同测试之间串流态极难调。后续 US-013 迁移脚本若用 `get_qdrant_client()` 一定记得在 fixture setUp/tearDown 里 `reset_singleton_for_tests()`。
  - Pattern discovered: **`get_client_from_config` 对 memory=True 不做探活**（直接 return）。原因：in-memory backend 的 `get_collections()` 必然成功但会触发上条 server-version warning；既然只在测试/单元场景用，省掉一次调用更干净。HTTP 模式才必须探活——PRD 的"立即 raise"语义专指远端连接失败。
  - Pattern discovered: **UP045（`Optional[X] → X | None`）对 Python 3.10+ 环境强制**。PRD 里写 `Optional[...]` 只是参照语义，落地时按 ruff 要求即可——这与 US-004 的 StrEnum vs `str, Enum` 是同一类"PRD 字面 vs lint gate"偏离，后续 M2 的 plan 里若出现 `Optional` 也按 `X | None` 写。
  - Gotchas 1：`get_qdrant_client(config)` 在 `_singleton` 已存在时**会忽略 config**——这是有意设计。若某测试用 `get_qdrant_client(QdrantConfig(memory=True))` 在之前的测试里缓存了 in-memory client，新测试即使传 HTTP config 也拿回来的是内存实例。**跨测试用 `reset_singleton_for_tests()` 清零**，不要依赖重复传参。
  - Gotchas 2：**port=1 不是通用"必定失败"端口**——大多数系统上闭，但如果有人启用 TCPMUX（RFC 1078）就不行。本测试用 `timeout=0.5` 兜底——就算三次握手偶发半连接，0.5s 内 QdrantClient 的 httpx 也会 timeout 后 raise ResponseHandlingException。M2 若用类似测法，timeout 不能设 >=5，否则本地拥塞时会误 block。
  - Gotchas 3：`QdrantClient` ctor 传 `host`/`port` 但**不接受 `timeout=None`**——必须传正数 float。若将来做"无限等待"语义，QdrantConfig.timeout 默认永远不要设为 None。
