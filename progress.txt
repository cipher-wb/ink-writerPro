## Codebase Patterns
- Schema 修改严格保持 `additionalProperties: false`：任何新顶层字段必须在 properties 显式声明，否则现存数据立即拒收。
- case_id pattern 容纳多前缀的方式：在主 alternation 段加分支（如 `LR(-[0-9]{4})?`），保留 `(M[0-9]+-)?` 与末尾 `[0-9]{4}$` 不动；每加一个前缀都要回归 `tests/case_library/test_schema_backward_compat.py::test_all_existing_cases_still_validate` 全 410 份。
- pytest.ini `testpaths` 是单行长串 — 新增目录追加在末尾即可，不要重排已有目录避免 diff 噪音；行后注释保持原样。
- ruff isort 要求：`from __future__ import annotations` 后空一行，再 stdlib，再空一行，再三方包；conftest/test 文件别忽略 `--fix` 自动修复。
- 测试用 `Draft202012Validator(load_schema(...))` fixture 模式：从 `tests/live_review/conftest.py` 取 `load_schema` fixture（按 schema 文件名加载），再实例化 validator；schema self-validity 用 `Draft202012Validator.check_schema(schema)`。
- Python 3.14 `datetime.utcnow()` 已 deprecated；新代码统一 `from datetime import UTC, datetime` + `datetime.now(UTC)`。ruff UP017 会提示。
- ID 分配复用 `case_library._id_alloc.allocate_case_id(cases_dir, prefix=...)`：传不同 prefix 自动隔离 counter file（`.id_alloc_<sanitized_prefix>.cnt`）+ glob 兜底防外部手工添加 yaml 让 counter 落后；并发安全（独占文件锁，4 worker spawn 0001..0004 无空洞已验证）。新 case 类型不需要再写 ID 分配器，写 `live_review.case_id.allocate_live_review_id` 这种薄封装即可。
- live-review 配置 dataclass 模式：`LiveReviewConfig` 是 frozen dataclass + 嵌套 `BatchConfig`/`InjectConfig`；`enabled=false` 在 `load_config` 内强制 `inject_into.{init,review}` 全 false（`& 与` 操作）— 这是关闭整个模块的 master switch，下游 inject point 务必查 `cfg.inject_into.<stage>` 而不是直接看 yaml。
- 资源路径必须锚定 `_REPO_ROOT = Path(__file__).resolve().parents[N]`：plan/示例里出现的 `Path("scripts/...")` / `Path("schemas/...")` 是相对 CWD 的脆弱写法，被 pytest fixture 调用时会随启动目录漂移。`ink_writer/live_review/*.py` 在 `parents[2]` 即 repo root；这条规矩适用所有 module-level 常量。
- `scripts/live-review/*.py` 子进程启动需自举 sys.path：脚本顶部加 `sys.path.insert(0, str(_REPO_ROOT))`（`_REPO_ROOT = Path(__file__).resolve().parents[2]`）才能 `from ink_writer.live_review.extractor import ...`，否则 `python3 scripts/live-review/foo.py` 直接启动会 ModuleNotFoundError。pytest 走 `pythonpath = .` 不需要这一步，但 subprocess test 必须靠脚本自举。
- ruff E402 的合规写法：windows stdio block 必须先于其他 import（构成 module level import），随后的 stdlib 导入 + 跨 `_REPO_ROOT` 注入后的 first-party 导入都加 `# noqa: E402`，与 `scripts/analyze_prose_directness.py` / `scripts/e2e_smoke_harness.py` 沿用同款。`# noqa: E402` 必须直接附在每行 import 后，行级粒度。
- BVID 字面值受 schema 约束 `^BV[A-Za-z0-9]+$`：PRD 字面占位（"BV-A"/"BV-B"等）含短横线不合法，fixture 必须用 `BV1AAA`/`BV1BBB`/`BV1CCC`/`BV1DDD`/`BV1EEE`（5 文件多份模式标准 ID 集，已固化在 mock_extract_5_files/raw_5_files）。
- Markdown 报告 parse 校验用 `MarkdownIt('commonmark').enable('table')`：默认 preset 不带 table 扩展；`'gfm-like'` preset 默认 enable linkify 但 linkify-it-py 未在依赖里会 raise ModuleNotFoundError。后续生成 markdown 报告（US-013/US-014）的 parse 测试沿用此组合。
- 脚本 stdout/stderr 分流：success path 写 stdout（如 `OK N files → path`），failure path 把详情 + report 写 stderr。测试断言可单边 `proc.stderr.contains` 而不被 success 干扰；CI/cron 也好用 grep 单独抓 OK 行。所有 `scripts/live-review/*.py` 沿用此约定。
- 单文件 vs 多份模式 argparse 共存：用一个触发参数（如 `--bvids`）作为分支开关，两组参数都 `required=False`，main 内手工校验 "single requires X/Y" / "many requires A/B"。比 mutually-exclusive group 灵活，能容纳各组多个必填字段。
- 批跑脚本 `--resume` + `--skip-failed` 组合：`--resume` 扫 `output_dir.glob("BV*.jsonl")` 取已完成 bvid set；循环内 `if bvid in done: continue`（log "skipped (resume)"）。`--skip-failed` 控制 `except` 分支 break 还是 continue —— `try/except Exception` 把 ExtractionError + mock 缺文件 + read_text 解码错都收编成一条 failure dict (`{bvid, error, traceback}`) 写进 `_failed.jsonl`。失败计数即使 break 退出也要写盘，方便用户审 traceback。退出码：失败且非 skip → 1；其余 → 0。
- `_failed.jsonl` 用 append 模式：多次 batch run 失败累积；测试时 tmp_path 每次新建可避免污染。`_failed.jsonl` 不应被 schema 校验脚本（US-005）当成普通 jsonl —— 后续 US-008 / US-013 等扫描 jsonl 时建议用 glob `BV*.jsonl` 而非 `*.jsonl` 排除 `_failed.jsonl`。
- 多份测试中"故意缺一个 mock 模拟失败"模式：fixture 目录放 N raw + (N-1) mock；缺失那个的 BV 在 mock_dir 取 path 时抛 FileNotFoundError，被 ExtractionError 捕获写进 _failed。需要"全 mock 齐"的兄弟测试用 fixture（fixture 名 `complete_mock_dir`）：拷贝 4 mock + 合成 1 minimal record 写入 tmp_path。这套 minimal record 用 `score=null / score_signal=unknown / verdict=unknown / comments=[]` 即可通过 schema 校验。
