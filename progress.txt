## Codebase Patterns
- Case dataclass 字段新增时必须同步：(1) `ink_writer/case_library/models.py` `Case.from_dict`/`to_dict`；(2) `schemas/case_schema.json` properties；(3) 现有 410 个 yaml 必须 backward-compatible（新字段 optional + from_dict 容错）。
- 新增测试目录必须加入 `pytest.ini` testpaths 单行（不会自动发现）；M5 已加 `tests/regression_tracker tests/meta_rule_emergence tests/dashboard tests/learn`。本批次新增 `tests/_compat` 同样需要加。
- 所有 `open()` 必带 `encoding="utf-8"`（CLAUDE.md Windows 兼容守则）。
- Python 入口可执行脚本须调 `runtime_compat.enable_windows_utf8_stdio()`。
- 项目用 python3（不是 python）；testpaths 已有 ~50 个目录单行；fixtures 在 `tests/<module>/conftest.py`。
- 跨平台文件锁原语：本批次 US-001 抽到 `ink_writer/_compat/locking.py`；以后任何需要文件锁的地方直接 import，不要再各模块复制 fcntl/msvcrt 代码。
- legacy case yaml 路径：`data/case_library/cases/CASE-*.yaml`（410 个）。
- `__main__.py` Windows UTF-8 引导块（参考 `ink_writer/creativity/__main__.py`）：先 `import os/sys` 把 `ink-writer/scripts` 加到 `sys.path`，try/except 调 `runtime_compat.enable_windows_utf8_stdio()`，然后才能 import 业务模块。pyproject 根 `exclude` 已豁免 ruff E402/I001 风险。
- `pyproject.toml` 顶层 `exclude` 排除了 `tests*`，所以 ruff 只 lint `ink_writer/**`；新增模块只需保证主代码 `ruff check` 全绿。
- 严重程度阶梯：`P3 → P2 → P1 → P0`，到 P0 不再升；累计次数靠 `len(recurrence_history)`。常量 `_SEVERITY_LADDER` 在 `ink_writer/case_library/store.py`。
- evidence_chain 检查器命中 case_id 至少 4 种载体：`checkers[].cases_hit` / `checkers[].cases_violated` / `checkers[].details.cases_hit` / `checkers[].details.cases_violated`；既支持 `["CASE-..."]` 字符串，也支持 `[{"case_id": "CASE-..."}]` dict。
- 多进程并发测试统一用 `mp.get_context("spawn")` 跨平台一致 — 见 `tests/case_library/test_id_alloc_concurrency.py` 与 `tests/rewrite_loop/test_dry_run_safety.py`。
- 原子写盘统一模式：`tempfile.mkstemp(prefix=..., suffix='.tmp', dir=...)` → `os.fdopen(fd,'w')` write+flush+fsync → `os.replace(tmp, target)`；except 路径 `tmp.unlink()` 兜底。本批次 US-008 强化 fd 安全。
- fail-loud 原则：上游 schema 漂移 / 锁失败 / 未知类型应显式抛错而非 silent pass — 参考 commit 47a6fe5 (Windows lock) 与本批次 US-006 (_serialize_case)。
