## Codebase Patterns
- 评估 evidence_chain 字段时优先扩展现有 dataclass（默认值保证向后兼容），不要新建并行 dataclass
- 写盘函数统一带 `encoding="utf-8"`，目录用 `Path.parent.mkdir(parents=True, exist_ok=True)`

- 测试 fixtures 走 tests/<area>/conftest.py，fixture 命名 `<area>_base_dir` + `sample_<area>_<scenario>`

- LLM 响应统一用 _extract_json* helper：先 try fence ```json ... ```，再退化为 raw "{ ... }" / "[ ... ]" 子串解析；失败抛 ValueError 让 except 捕获重试

- agent spec 落 ink-writer/agents/<name>-checker.md，frontmatter 含 name/description/tools=Read/model=inherit；包含来源 spec/PRD、Python 调用示例、字段表、阻断行为、与下游衔接段

  - genre_novelty 的 score 反向（与 M3 章节级 checker 同向）：低 score = 高相似 = 阻断；spec §3.1 是 1.0 - max(sim)，不要写成 mean(sim)
  - top5 array 输出时显式按 similarity 降序排序后 [:5]：LLM 偶尔会返回多于/少于 5 条或排序错乱，用 `enriched.sort(key=lambda x: x["similarity"], reverse=True)` 兜底，确保 top5_similar[0] 即 max

  - max_retries 默认 2（M4 上游层 spec），与 M3 章节级 checker 默认 3 不同——抄 protagonist_agency 时记得改默认值
