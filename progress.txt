## Codebase Patterns
- IndexManager auto-creates all DB tables on init; never create tables manually in fixtures — let IndexManager.__init__ handle schema, then INSERT test data
- E402 lint warnings are expected for scripts using sys.path manipulation before imports (project_locator, data_modules)
- DataModulesConfig.from_project_root(path) expects `.ink/` dir with `state.json` under project_root
- Review metrics are stored in review_metrics table with JSON-encoded dimension_scores, severity_counts, review_payload_json
- chapter_reading_power table stores per-chapter hook/micropayoff data; tool_call_stats has limited columns (no duration_ms/tokens_used)
- pytest.ini testpaths: `ink-writer/scripts/data_modules/tests tests/baseline tests/audit tests/hooks tests/pacing tests/emotion tests/style_rag`; pythonpath includes `scripts`
- Agent specs are markdown with YAML frontmatter (name/description/tools/model); IO sections labeled "输入"/"输出格式"
- Import cycle detection works via pure Python AST + DFS; no need for networkx dependency
- 14 agents in `ink-writer/agents/`, 1 in `agents/ink-writer/` (editor-wisdom-checker)
- benchmark/corpus_index.json has 117 books; use `collections` field for ranking, `dir` field for filesystem lookup
- benchmark/config.py has SCENE_TYPE_KEYWORDS / EMOTION_KEYWORDS — reuse for text analysis instead of reinventing
- Hook types taxonomy (5 core): crisis, mystery, emotion, choice, desire — aligned with reader-pull-checker agent
- Position types: open (chapter opening), mid (mid-chapter tension), cliff (chapter-end cliffhanger)
- Genre hook templates in GENRE_HOOK_TEMPLATES cover 13 genres; each genre has 5 templates across different hook types
- data/hook_patterns.json is the canonical hook pattern library; regenerate via `python3 scripts/mine_hook_patterns.py`
- Chapter outline schema is Markdown-based (not JSON); defined in ink-plan/SKILL.md lines 448-468
- hook_contract field format: `类型=X | 兑现锚点=第M章 | 兑现摘要=Y` — parsed by ink-writer/scripts/hook_contract.py
- Outline backfill migration: `python3 scripts/patch_outline_hook_contract.py --project-root <path>` — backs up originals before patching
- migrate.py CURRENT_SCHEMA_VERSION=8; v7→v8 adds hook_contract_config to state.json
- reader-pull hook retry gate: `ink_writer.reader_pull.hook_retry_gate.run_hook_gate()` — mirrors editor_wisdom/review_gate.py pattern
- fix_prompt_builder: `VIOLATION_FIX_TEMPLATES` maps violation IDs to Chinese fix instructions; `normalize_checker_output()` handles both old and new output formats
- config/reader-pull.yaml: score_threshold=70, golden_three_threshold=80, max_retries=2
- polish-agent accepts `hook_fix_prompt` field for targeted hook/追读力 repairs (Step 1.5 in execution flow)
- ink-write Step 3.6 is the hook retry gate; runs after Step 3.5 harness gate, before Step 4
- high_point_scheduler: `ink_writer.pacing.high_point_scheduler.schedule_high_point()` — proactive cool-point recipe per chapter
- Cool-point type taxonomy (6 types): face_slap, hidden_strength, level_up_kill, authority_challenge, villain_fail, sweet_surprise
- Intensity levels: minor (小爽点), combo (组合爽点), milestone (里程碑爽点) — from cool-points-guide.md
- config/high-point-scheduler.yaml: max_consecutive_no_hp=2, combo_window=5, milestone_window=12, climax_zone_start=0.85
- ink-plan Step 2.5 is high-point scheduling; Step 2.6 is cool-point execution script (was Step 2.5)
- Chapter outline template `爽点配方` field format: `类型=X | 强度=Y | 兑现窗口=Z章`
