## Codebase Patterns
(承接自前轮 ralph runs，保留跨 PRD 通用条目。逐轮新增追加到下方。)

- **UTF-8 编码硬约束**：所有 `open()` / `Path.read_text()` / `Path.write_text()` 必须带 `encoding="utf-8"`。scripts/fix_utf8_encoding.py 可批量修复；scripts/audit_cross_platform.py 是 9 类风险单一扫描入口。
- **零回归守卫 baseline**：v21.0.0 发版后 `pytest --no-cov` 全量 baseline = **3206 passed / 23 skipped / 0 failed**。本轮每个 US 完成前必须跑全量无新增失败。
- **pytest.ini testpaths 同步**：新增的 `tests/核心模块/子目录/` 必须追加到 `pytest.ini:testpaths`，否则默认收集会漏掉。
- **pytest 跨平台标记**（v21 起）：用 `@pytest.mark.windows` / `@pytest.mark.mac` 而非 `@pytest.mark.skipif(sys.platform != "...")`。`conftest.py` autoskip；仓库红线禁止回退到 skipif。
- **ProjectConfig 读取约定**：章节正文文件命名/后缀由 `ProjectConfig` 决定；新代码不硬编码命名。
- **ralph runtime 文件不入 commit**：`.claude/.ink-current-project`、`scripts/ralph/.last-branch` 是 runtime 状态；归档副本在 `scripts/ralph/archive/` 下受版本控制。
- **kwarg-only 向后兼容**：给既有 public 函数新增参数时放到 `*` 之后（kwarg-only），避免破坏生产/测试调用方的位置参数假设。
- **runtime_compat 原语集中复用**：`ink-writer/scripts/runtime_compat.py` 已暴露 6 个 helper（`enable_windows_utf8_stdio` / `normalize_windows_path` / `set_windows_proactor_policy` / `_has_symlink_privilege` / `find_python_launcher` / `safe_symlink`）——新增跨平台 helper 统一放这里。
- **Mac 字节级一致承诺**（v19 起）：所有 Windows 特化代码走 `if sys.platform == "win32":` 分支，`.sh` 文件字节不变——本轮修复须严格保留。
- **COMPLETE 信号锚定**（v21 US-011 起）：`scripts/ralph/ralph.sh` 的 COMPLETE 检测用 `tail -n 50 | grep -qE '^[[:space:]]*<promise>COMPLETE</promise>[[:space:]]*$'`——讨论里引用这个字符串不会触发早停。本轮只需按正常方式在最终总结里输出 `<promise>COMPLETE</promise>` 独占一行。
- **jieba posseg 延迟导入**（US-001 起）：scripts 中的 jieba.posseg 加载 ~200ms；用 `_lazy_jieba()` helper 放到函数体而非顶部 import。模块导入期不触发，避免 pytest 收集期加重。
- **中文对话 unicode 显式写**（US-001 起）：源码里"「」""" 这些弯引号字符容易被编辑器/终端折叠成直引号，导致运行时检测失败。统一用 `"\u201c"`/`"\u201d"` 等显式 unicode 转义。
- **ruff E402 规避**：scripts/ 入口在 `if sys.platform == "win32":` runtime_compat setup 之后才能 import 其他模块，统一加 `# noqa: E402` 到那批 late imports，`from __future__ import annotations` 后置不变。
