# 输出要求

严格输出一段 JSON（不得包裹在 markdown 代码块中），形如：

{{
  "rule_scores": {{
    "<rule_id>": <0-1 之间的浮点数>,
    ...
  }},
  "case_evaluation": [
    {{
      "case_id": "<CASE-XXXX-XXXX>",
      "addressed": true|false,
      "evidence": "<引用本章中的具体片段或一句话说明为什么 addressed>"
    }},
    ...
  ],
  "notes": "<简短自评说明，可空字符串>"
}}

约束：
- rule_scores 的 key 必须使用 injected_rules 给出的 rule_id；漏给的将按 0 计入。
- case_evaluation 的每个 case_id 必须出现在 applicable_cases 中；漏给的将按
  addressed=False 处理（保守判定）。
- 所有评分必须以正文证据为准，禁止臆测正文未出现的内容。
- 不要输出除 JSON 之外的其他文本。
